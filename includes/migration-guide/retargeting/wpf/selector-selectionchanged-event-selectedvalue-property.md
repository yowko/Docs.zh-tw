|<span data-ttu-id="69750-102">詳細資料</span><span class="sxs-lookup"><span data-stu-id="69750-102">Details</span></span>|<span data-ttu-id="69750-103">從 .Net Framework 4.7.1 開始，<xref:System.Windows.Controls.Primitives.Selector> 在其選取項目變更時，一律會先更新其 <xref:System.Windows.Controls.Primitives.Selector.SelectedValue%2A> 屬性的值，再引發 <xref:System.Windows.Controls.Primitives.Selector.SelectionChanged> 事件。</span><span class="sxs-lookup"><span data-stu-id="69750-103">Starting with the .Net Framework 4.7.1, a <xref:System.Windows.Controls.Primitives.Selector> always updates the value of its <xref:System.Windows.Controls.Primitives.Selector.SelectedValue%2A> property before raising the <xref:System.Windows.Controls.Primitives.Selector.SelectionChanged> event, when its selection changes.</span></span> <span data-ttu-id="69750-104">這會讓 SelectedValue 屬性與其他選取項目屬性 (<xref:System.Windows.Controls.Primitives.Selector.SelectedItem%2A> 和 <xref:System.Windows.Controls.Primitives.Selector.SelectedIndex%2A>) 一致，在引發事件之前會更新這些選取項目屬性。在 .NET Framework 4.7 和舊版中，在大部分情況下，更新 SelectedValue 會發生在事件之前，但如果選取項目變更是由於變更 <xref:System.Windows.Controls.Primitives.Selector.SelectedValue%2A> 屬性所造成，則會發生在事件之後。</span><span class="sxs-lookup"><span data-stu-id="69750-104">This makes the SelectedValue property consistent with the other selection properties (<xref:System.Windows.Controls.Primitives.Selector.SelectedItem%2A> and <xref:System.Windows.Controls.Primitives.Selector.SelectedIndex%2A>), which are updated before raising the event.In the .NET Framework 4.7 and earlier versions, the update to SelectedValue happened before the event in most cases, but it happened after the event if the selection change was caused by changing the <xref:System.Windows.Controls.Primitives.Selector.SelectedValue%2A> property.</span></span>|
