# <a name="dynamically-loading-and-using-types"></a><span data-ttu-id="65503-102">動態載入和使用類型</span><span class="sxs-lookup"><span data-stu-id="65503-102">Dynamically Loading and Using Types</span></span>
<span data-ttu-id="65503-103">反映會提供語言編譯器所使用的基礎結構，例如 [!INCLUDE[vbprvbext](../../../includes/vbprvbext-md.md)] 和 JScript，以實作隱含晚期繫結。</span><span class="sxs-lookup"><span data-stu-id="65503-103">Reflection provides infrastructure used by language compilers such as [!INCLUDE[vbprvbext](../../../includes/vbprvbext-md.md)] and JScript to implement implicit late binding.</span></span> <span data-ttu-id="65503-104">繫結是尋找對應至唯一指定的類型宣告 (也就是實作) 的程序。</span><span class="sxs-lookup"><span data-stu-id="65503-104">Binding is the process of locating the declaration (that is, the implementation) that corresponds to a uniquely specified type.</span></span> <span data-ttu-id="65503-105">當此程序發生在執行階段，而不是在編譯時期時，它稱為晚期繫結。</span><span class="sxs-lookup"><span data-stu-id="65503-105">When this process occurs at run time rather than at compile time, it is called late binding.</span></span> [!INCLUDE[vbprvblong](../../../includes/vbprvblong-md.md)] <span data-ttu-id="65503-106">可讓您在程式碼內使用隱含晚期繫結；Visual Basic 編譯器會呼叫 helper 方法，它會使用反映來取得物件類型。</span><span class="sxs-lookup"><span data-stu-id="65503-106">allows you to use implicit late binding in your code; the Visual Basic compiler calls a helper method that uses reflection to obtain the object type.</span></span> <span data-ttu-id="65503-107">傳遞至 helper 方法的引數會導致在執行階段叫用適當的方法。</span><span class="sxs-lookup"><span data-stu-id="65503-107">The arguments passed to the helper method cause the appropriate method to be invoked at run time.</span></span> <span data-ttu-id="65503-108">這些引數是在其上叫用方法的執行個體 (物件)、被叫用方法的名稱 (字串)，以及傳遞給被叫用方法的引數 (物件陣列)。</span><span class="sxs-lookup"><span data-stu-id="65503-108">These arguments are the instance (an object) on which to invoke the method, the name of the invoked method (a string), and the arguments passed to the invoked method (an array of objects).</span></span>  
  
 <span data-ttu-id="65503-109">在下列範例中，Visual Basic 編譯器會隱含地使用反映，對一個在編譯階段不知道其類型的物件呼叫方法。</span><span class="sxs-lookup"><span data-stu-id="65503-109">In the following example, the Visual Basic compiler uses reflection implicitly to call a method on an object whose type is not known at compile time.</span></span> <span data-ttu-id="65503-110">**HelloWorld** 類別具有 **PrintHello** 方法，它會列印出 "Hello World" 並串連傳遞給 **PrintHello** 方法的部分文字。</span><span class="sxs-lookup"><span data-stu-id="65503-110">A **HelloWorld** class has a **PrintHello** method that prints out "Hello World" concatenated with some text that is passed to the **PrintHello** method.</span></span> <span data-ttu-id="65503-111">在此範例中呼叫的 **PrintHello** 方法實際上是 <xref:System.Type.InvokeMember%2A?displayProperty=nameWithType>；Visual Basic 程式碼允許叫用 **PrintHello** 方法，就彷彿已在編譯階段知道物件 (helloObj) 的類型 (早期繫結)，而不是執行階段知道 (晚期繫結)。</span><span class="sxs-lookup"><span data-stu-id="65503-111">The **PrintHello** method called in this example is actually a <xref:System.Type.InvokeMember%2A?displayProperty=nameWithType>; the Visual Basic code allows the **PrintHello** method to be invoked as if the type of the object (helloObj) were known at compile time (early binding) rather than at run time (late binding).</span></span>  
  
```  
Imports System  
Module Hello  
    Sub Main()  
        ' Sets up the variable.  
        Dim helloObj As Object  
        ' Creates the object.  
        helloObj = new HelloWorld()  
        ' Invokes the print method as if it was early bound  
        ' even though it is really late bound.  
        helloObj.PrintHello("Visual Basic Late Bound")  
    End Sub  
End Module  
```
