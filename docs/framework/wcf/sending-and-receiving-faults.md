 <span data-ttu-id="e1111-146">就算是本機物件，處理物件也可能引發或遮罩在回收處理序期間所發生的例外狀況。</span><span class="sxs-lookup"><span data-stu-id="e1111-146">Even with local objects disposing the object can either raise or mask exceptions that occur during the recycling process.</span></span> <span data-ttu-id="e1111-147">當您使用 [!INCLUDE[indigo2](../../../includes/indigo2-md.md)] 用戶端物件時，也會發生類似的情況。</span><span class="sxs-lookup"><span data-stu-id="e1111-147">Something similar can occur when you use [!INCLUDE[indigo2](../../../includes/indigo2-md.md)] client objects.</span></span> <span data-ttu-id="e1111-148">當您呼叫作業時，事實上您是透過已建立的連線來傳送訊息。</span><span class="sxs-lookup"><span data-stu-id="e1111-148">When you call operations you are sending messages over an established connection.</span></span> <span data-ttu-id="e1111-149">如果連線無法完全關閉，或是已經關閉，則關閉通道會擲回例外狀況，就算所有作業都正常傳回也是一樣。</span><span class="sxs-lookup"><span data-stu-id="e1111-149">Closing the channel can throw exceptions if the connection cannot be cleanly closed or is already closed, even if all the operations returned properly.</span></span>  
  
 <span data-ttu-id="e1111-150">一般來說，用戶端物件通道可透過下列其中一種方式來關閉：</span><span class="sxs-lookup"><span data-stu-id="e1111-150">Typically, client object channels are closed in one of the following ways:</span></span>  
  
-   <span data-ttu-id="e1111-151">當 [!INCLUDE[indigo2](../../../includes/indigo2-md.md)] 用戶端物件已回收時。</span><span class="sxs-lookup"><span data-stu-id="e1111-151">When the [!INCLUDE[indigo2](../../../includes/indigo2-md.md)] client object is recycled.</span></span>
