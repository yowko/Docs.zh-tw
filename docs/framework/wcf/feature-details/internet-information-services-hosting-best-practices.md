## <a name="state-management-and-process-recycling"></a><span data-ttu-id="40acb-115">狀態管理與處理序回收</span><span class="sxs-lookup"><span data-stu-id="40acb-115">State Management and Process Recycling</span></span>

> <span data-ttu-id="40acb-119">WCF 用來進行訊息層可靠性和安全性的通訊協定會使用 volatile 記憶體內部狀態。</span><span class="sxs-lookup"><span data-stu-id="40acb-119">The protocols WCF uses for message-layer reliability and security make use of the volatile in-memory state.</span></span> <span data-ttu-id="40acb-120">WCF 可靠會話和安全性會話可能會因為應用程式回收而非預期地終止。</span><span class="sxs-lookup"><span data-stu-id="40acb-120">WCF reliable sessions and security sessions may terminate unexpectedly due to application recycles.</span></span> <span data-ttu-id="40acb-121">使用這些通訊協定的 IIS 裝載應用程式, 應該取決於 WCF 提供的工作階段金鑰, 以關聯應用層狀態 (例如, 應用層結構或自訂相互關聯標頭) 或停用。裝載應用程式的 IIS 進程回收。</span><span class="sxs-lookup"><span data-stu-id="40acb-121">IIS-hosted applications that make use of these protocols should either depend on something other than the WCF-provided session key for correlating application-layer state (for example, an application-layer construct or custom correlation header) or disable IIS process recycling for the hosted application.</span></span>
