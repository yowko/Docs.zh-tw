## <a name="hop-counts"></a><span data-ttu-id="2c87b-110">躍點計數</span><span class="sxs-lookup"><span data-stu-id="2c87b-110">Hop Counts</span></span>  
 <span data-ttu-id="2c87b-111">`PeerHopCount` 的概念類似於 IP 通訊協定中使用的 TTL (存留時間)。</span><span class="sxs-lookup"><span data-stu-id="2c87b-111">The concept of `PeerHopCount` is similar to TTL (Time-To-Live) used in IP protocol.</span></span> <span data-ttu-id="2c87b-112">`PeerHopCount` 的值會繫結到訊息執行個體，並會指定在捨棄訊息前應該轉送幾次訊息。</span><span class="sxs-lookup"><span data-stu-id="2c87b-112">The value of `PeerHopCount` is tied to a message instance, and it specifies how many times a message should be forwarded before being dropped.</span></span> <span data-ttu-id="2c87b-113">每當對等通道用戶端收到訊息時，用戶端會檢查訊息是否有指定 `PeerHopCount`。</span><span class="sxs-lookup"><span data-stu-id="2c87b-113">Each time a message is received by a Peer Channel client, the client examines the message to see if `PeerHopCount` is specified.</span></span> <span data-ttu-id="2c87b-114">如果有指定的話，則用戶端會將躍點計數值減一後，才轉送訊息給鄰近節點。</span><span class="sxs-lookup"><span data-stu-id="2c87b-114">If it is specified, then the client decrements the hop count value by one before forwarding the message to neighboring nodes.</span></span> <span data-ttu-id="2c87b-115">當用戶端所收到訊息的躍點計數值為零時，用戶端會處理該訊息，但不會將訊息轉送給鄰近節點。</span><span class="sxs-lookup"><span data-stu-id="2c87b-115">When a client receives a message with a hop count value of zero, the client processes the message, but does not forward the message to neighbors.</span></span>  
  
 <span data-ttu-id="2c87b-116">只要將 `PeerHopCount` 當做屬性 (Attribute) 加入至訊息類別實作中適當的屬性 (Property) 或欄位中，即可將躍點計數加入至訊息。</span><span class="sxs-lookup"><span data-stu-id="2c87b-116">Hop count may be added to a message by adding `PeerHopCount` as an attribute to the applicable property or field in the implementation of the message class.</span></span> <span data-ttu-id="2c87b-117">您可以在傳送訊息至網狀結構之前，先將此屬性設定為特定值。</span><span class="sxs-lookup"><span data-stu-id="2c87b-117">You can set this to a specific value before sending the message to the mesh.</span></span> <span data-ttu-id="2c87b-118">因此，必要時您可以使用躍點計數來限制訊息在 mesh 間的散發，或許可以避免不必要的重複訊息。</span><span class="sxs-lookup"><span data-stu-id="2c87b-118">In this manner, you can use hop count to limit distribution of messages throughout the mesh when necessary, potentially avoiding unnecessary message duplication.</span></span> <span data-ttu-id="2c87b-119">當 mesh 包含大量的多餘資料，或者要傳送訊息給最近的鄰居或少許躍點內的鄰居時，這個方法會特別有用。</span><span class="sxs-lookup"><span data-stu-id="2c87b-119">This is useful in cases where the mesh contains a high amount of redundant data, or for sending a message to immediate neighbors, or neighbors within a few hops.</span></span>
