 <span data-ttu-id="70d47-114">A*資料包*通道是所有的訊息沒有相互關聯的通道。</span><span class="sxs-lookup"><span data-stu-id="70d47-114">A *datagram* channel is a channel in which all messages are uncorrelated.</span></span> <span data-ttu-id="70d47-115">使用資料包通道時，如果輸入或輸出作業失敗，下一個作業通常不會受到影響，而且還可以重複使用同一個通道。</span><span class="sxs-lookup"><span data-stu-id="70d47-115">With a datagram channel, if an input or output operation fails, the next operation is typically unaffected, and the same channel can be reused.</span></span> <span data-ttu-id="70d47-116">基於這點，資料包通道通常不會發生錯誤。</span><span class="sxs-lookup"><span data-stu-id="70d47-116">Because of this, datagram channels typically do not fault.</span></span>  
  
 <span data-ttu-id="70d47-117">*工作階段*通道，不過，會使用連線到另一個端點的通道。</span><span class="sxs-lookup"><span data-stu-id="70d47-117">*Sessionful* channels, however, are channels with a connection to the other endpoint.</span></span> <span data-ttu-id="70d47-118">某一端工作階段中的訊息永遠都會與另一端的相同工作階段相關聯。</span><span class="sxs-lookup"><span data-stu-id="70d47-118">Messages in a session on one side are always correlated with the same session on the other side.</span></span> <span data-ttu-id="70d47-119">此外，工作階段中的參與者雙方都必須同意其對話需求已被滿足，這樣工作階段才能算是成功。</span><span class="sxs-lookup"><span data-stu-id="70d47-119">In addition, both participants in a session must agree that the requirements of their conversation were met for that session to be considered successful.</span></span> <span data-ttu-id="70d47-120">如果它們不同意，工作階段通道可能會發生錯誤。</span><span class="sxs-lookup"><span data-stu-id="70d47-120">If they cannot agree, the sessionful channel may fault.</span></span>
