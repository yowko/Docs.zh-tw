 <span data-ttu-id="7ee10-173">除了處理動態字典索引鍵之外，還會以獨特的方式來接收緩衝的工作階段訊息。</span><span class="sxs-lookup"><span data-stu-id="7ee10-173">In addition to processing dynamic dictionary keys, buffered sessionful messages are received in a unique way.</span></span> <span data-ttu-id="7ee10-174">二進位編碼器並非在文件上建立讀取器，然後加以處理，而是使用內部的 `MessagePatterns` 類別來拆解二進位資料流。</span><span class="sxs-lookup"><span data-stu-id="7ee10-174">Instead of creating a reader over the document and processing it, the binary encoder uses the internal `MessagePatterns` class to deconstruct the binary stream.</span></span> <span data-ttu-id="7ee10-175">其概念是，大部分的訊息有一組特定的顯示在以特定順序時由 WCF 所產生的標頭。</span><span class="sxs-lookup"><span data-stu-id="7ee10-175">The idea is that most messages have a certain set of headers that show up in a certain order when generated by WCF.</span></span> <span data-ttu-id="7ee10-176">模式系統會根據其需求分解訊息。</span><span class="sxs-lookup"><span data-stu-id="7ee10-176">The pattern system breaks the message apart based on what it expects.</span></span> <span data-ttu-id="7ee10-177">如果分解成功，這個系統就會將 <xref:System.ServiceModel.Channels.MessageHeaders> 物件初始化，而不剖析 XML。</span><span class="sxs-lookup"><span data-stu-id="7ee10-177">If it is successful, it initializes a <xref:System.ServiceModel.Channels.MessageHeaders> object without parsing the XML.</span></span> <span data-ttu-id="7ee10-178">如果失敗，則退而求其次，使用標準的方法。</span><span class="sxs-lookup"><span data-stu-id="7ee10-178">If not, it falls back to the standard method.</span></span>  
  
### <a name="mtom-encoding"></a><span data-ttu-id="7ee10-179">MTOM 編碼方式</span><span class="sxs-lookup"><span data-stu-id="7ee10-179">MTOM Encoding</span></span>  
 <span data-ttu-id="7ee10-180"><xref:System.ServiceModel.Channels.MtomMessageEncodingBindingElement> 類別具有另一個名為 <xref:System.ServiceModel.Channels.MtomMessageEncodingBindingElement.MaxBufferSize%2A> 的組態屬性。</span><span class="sxs-lookup"><span data-stu-id="7ee10-180">The <xref:System.ServiceModel.Channels.MtomMessageEncodingBindingElement> class has an additional configuration property called <xref:System.ServiceModel.Channels.MtomMessageEncodingBindingElement.MaxBufferSize%2A>.</span></span> <span data-ttu-id="7ee10-181">這個屬性可以設定允許繫結項目在讀取訊息過程中緩衝資料數量的上限。</span><span class="sxs-lookup"><span data-stu-id="7ee10-181">This places an upper bound on how much data it is allowed to buffer during the process of reading a message.</span></span> <span data-ttu-id="7ee10-182">可能需要將 XML 資訊集 (Infoset) 或其他 MIME 部分加以緩衝，才能將所有 MIME 部分重組成單一訊息。</span><span class="sxs-lookup"><span data-stu-id="7ee10-182">The XML Information Set (Infoset), or other MIME parts, may need to be buffered to reassemble all the MIME parts into a single message.</span></span>
