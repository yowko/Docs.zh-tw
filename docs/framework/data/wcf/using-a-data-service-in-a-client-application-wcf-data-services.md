 <span data-ttu-id="7041d-110">OData 有助於保證公開 OData 摘要的服務與取用 OData 摘要的應用程式之間的互通性。</span><span class="sxs-lookup"><span data-stu-id="7041d-110">OData helps guarantee interoperability between services that expose OData feeds and applications that consume OData feeds.</span></span> <span data-ttu-id="7041d-111">應用程式會藉由傳送特定 HTTP 動作的要求訊息，以及處理應該執行動作之實體資源的 URI，來存取及變更以 OData 為基礎的服務中的資料。</span><span class="sxs-lookup"><span data-stu-id="7041d-111">Applications access and change data in an OData-based service by sending request messages of a specific HTTP action and with a URI that addresses an entity resource against which the action should be performed.</span></span> <span data-ttu-id="7041d-112">當必須提供實體資料時，會在訊息本文中以特定編碼的裝載形式提供。</span><span class="sxs-lookup"><span data-stu-id="7041d-112">When entity data must be supplied, it is supplied as a specifically encoded payload in the body of the message.</span></span>  
  
### <a name="http-actions"></a><span data-ttu-id="7041d-113">HTTP 動作</span><span class="sxs-lookup"><span data-stu-id="7041d-113">HTTP Actions</span></span>  
 <span data-ttu-id="7041d-114">OData 支援下列 HTTP 動作，在定址資源代表的實體資料上執行建立、讀取、更新和刪除作業：</span><span class="sxs-lookup"><span data-stu-id="7041d-114">OData supports the following HTTP actions to perform create, read, update, and delete operations on the entity data that the addressed resource represents:</span></span>  
  
- <span data-ttu-id="7041d-115">**HTTP GET** -這是從瀏覽器存取資源時的預設動作。</span><span class="sxs-lookup"><span data-stu-id="7041d-115">**HTTP GET** - This is the default action when a resource is accessed from a browser.</span></span> <span data-ttu-id="7041d-116">要求訊息中不會提供任何承載，同時會以包含所要求之資料的承載傳回回應方法。</span><span class="sxs-lookup"><span data-stu-id="7041d-116">No payload is supplied in the request message, and a response method with a payload that contains the requested data is returned.</span></span>  
  
- <span data-ttu-id="7041d-117">**HTTP POST**會將新的實體資料插入提供的資源中。</span><span class="sxs-lookup"><span data-stu-id="7041d-117">**HTTP POST** - Inserts new entity data into the supplied resource.</span></span> <span data-ttu-id="7041d-118">要插入的資料會於要求訊息的承載中提供。</span><span class="sxs-lookup"><span data-stu-id="7041d-118">Data to be inserted is supplied in the payload of the request message.</span></span> <span data-ttu-id="7041d-119">回應訊息的裝載包含新建立之實體的資料。</span><span class="sxs-lookup"><span data-stu-id="7041d-119">The payload of the response message contains the data for the newly created entity.</span></span> <span data-ttu-id="7041d-120">其中包括任何自動產生的索引鍵值。</span><span class="sxs-lookup"><span data-stu-id="7041d-120">This includes any autogenerated key values.</span></span> <span data-ttu-id="7041d-121">標頭亦包含定址新實體資源的 URI。</span><span class="sxs-lookup"><span data-stu-id="7041d-121">The header also contains the URI that addresses the new entity resource.</span></span>  
  
- <span data-ttu-id="7041d-122">**HTTP 刪除**-刪除指定資源所代表的實體資料。</span><span class="sxs-lookup"><span data-stu-id="7041d-122">**HTTP DELETE** - Deletes the entity data that the specified resource represents.</span></span> <span data-ttu-id="7041d-123">要求或回應訊息中不會出現任何承載。</span><span class="sxs-lookup"><span data-stu-id="7041d-123">A payload is not present in the request or response messages.</span></span>  
  
- <span data-ttu-id="7041d-124">**HTTP PUT** -以要求訊息的承載中提供的新資料取代所要求資源中的現有實體資料。</span><span class="sxs-lookup"><span data-stu-id="7041d-124">**HTTP PUT** - Replaces existing entity data at the requested resource with new data that is supplied in the payload of the request message.</span></span>  
  
- <span data-ttu-id="7041d-125">**HTTP merge** -由於只為了變更實體資料而執行刪除時，如果只是在資料來源中插入，所以 OData 引進了新的 HTTP MERGE 動作。</span><span class="sxs-lookup"><span data-stu-id="7041d-125">**HTTP MERGE** - Because of inefficiencies in executing a delete followed by an insert in the data source just to change entity data, OData introduces a new HTTP MERGE action.</span></span> <span data-ttu-id="7041d-126">要求訊息的承載包含在定址實體資源中必須變更的屬性。</span><span class="sxs-lookup"><span data-stu-id="7041d-126">The payload of the request message contains the properties that must be changed on the addressed entity resource.</span></span> <span data-ttu-id="7041d-127">由於 HTTP 規格中未定義 HTTP MERGE，因此可能需要額外的處理，才能透過非 OData 感知伺服器來路由傳送 HTTP MERGE 要求。</span><span class="sxs-lookup"><span data-stu-id="7041d-127">Because HTTP MERGE is not defined in the HTTP specification, it may require additional processing to route a HTTP MERGE request through non-OData aware servers.</span></span>  
  
 <span data-ttu-id="7041d-128">如需詳細資訊，請參閱[OData：作業](https://www.odata.org/documentation/odata-version-2-0/operations/)。</span><span class="sxs-lookup"><span data-stu-id="7041d-128">For more information, see [OData: Operations](https://www.odata.org/documentation/odata-version-2-0/operations/).</span></span>
