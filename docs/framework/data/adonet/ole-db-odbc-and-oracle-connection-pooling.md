## <a name="connection-pooling-for-oracleclient"></a><span data-ttu-id="7df5d-118">OracleClient 的連接共用</span><span class="sxs-lookup"><span data-stu-id="7df5d-118">Connection Pooling for OracleClient</span></span>

### <a name="pool-creation-and-assignment"></a><span data-ttu-id="7df5d-121">集區的建立及指派</span><span class="sxs-lookup"><span data-stu-id="7df5d-121">Pool Creation and Assignment</span></span>  
 <span data-ttu-id="7df5d-122">當開啟連接時，會依據將集區與連接中連接字串相關聯的精確比對演算法，建立連接集區。</span><span class="sxs-lookup"><span data-stu-id="7df5d-122">When a connection is opened, a connection pool is created based on an exact matching algorithm that associates the pool with the connection string in the connection.</span></span> <span data-ttu-id="7df5d-123">每個連接集區與不同的連接字串相關聯。</span><span class="sxs-lookup"><span data-stu-id="7df5d-123">Each connection pool is associated with a distinct connection string.</span></span> <span data-ttu-id="7df5d-124">開啟新連接時，如果連接字串與現有集區並不完全相符，則會建立新集區。</span><span class="sxs-lookup"><span data-stu-id="7df5d-124">When a new connection is opened, if the connection string is not an exact match to an existing pool, a new pool is created.</span></span>  
  
 <span data-ttu-id="7df5d-125">一旦建立，便無法損毀連接集區，直到作用中處理序結束為止。</span><span class="sxs-lookup"><span data-stu-id="7df5d-125">Once created, connection pools are not destroyed until the active process ends.</span></span> <span data-ttu-id="7df5d-126">維護非作用中或空的集區所耗用的系統資源極少。</span><span class="sxs-lookup"><span data-stu-id="7df5d-126">Maintaining inactive or empty pools uses very few system resources.</span></span>  
  
### <a name="connection-addition"></a><span data-ttu-id="7df5d-127">加入連接</span><span class="sxs-lookup"><span data-stu-id="7df5d-127">Connection Addition</span></span>  
 <span data-ttu-id="7df5d-128">針對每個唯一連接字串可建立連接集區。</span><span class="sxs-lookup"><span data-stu-id="7df5d-128">A connection pool is created for each unique connection string.</span></span> <span data-ttu-id="7df5d-129">建立集區時，會建立多個連接物件並加入集區，以滿足最小集區大小需求。</span><span class="sxs-lookup"><span data-stu-id="7df5d-129">When a pool is created, multiple connection objects are created and added to the pool so that the minimum pool size requirement is satisfied.</span></span> <span data-ttu-id="7df5d-130">視需要將連接加入集區，不超過最大集區大小。</span><span class="sxs-lookup"><span data-stu-id="7df5d-130">Connections are added to the pool as needed, up to the maximum pool size.</span></span>  
  
 <span data-ttu-id="7df5d-131">當要求 <xref:System.Data.OracleClient.OracleConnection> 物件時，若有可用的連接，則會從集區取得該物件。</span><span class="sxs-lookup"><span data-stu-id="7df5d-131">When an <xref:System.Data.OracleClient.OracleConnection> object is requested, it is obtained from the pool if a usable connection is available.</span></span> <span data-ttu-id="7df5d-132">若要可用，連接目前必須為未使用，並具有相符的交易內容，或未與任何交易內容關聯，同時具有到伺服器的有效連結。</span><span class="sxs-lookup"><span data-stu-id="7df5d-132">To be usable, the connection must currently be unused, have a matching transaction context or not be associated with any transaction context, and have a valid link to the server.</span></span>  
  
 <span data-ttu-id="7df5d-133">如果已達到最大集區大小，但仍沒有可用的連接，則會將要求排入佇列。</span><span class="sxs-lookup"><span data-stu-id="7df5d-133">If the maximum pool size has been reached and no usable connection is available, the request is queued.</span></span> <span data-ttu-id="7df5d-134">連接共用器會在連接釋放回集區後，重新配置連接以滿足這些要求。</span><span class="sxs-lookup"><span data-stu-id="7df5d-134">The connection pooler satisfies these requests by reallocating connections as they are released back into the pool.</span></span> <span data-ttu-id="7df5d-135">連接關閉或處置時，會被釋放回集區。</span><span class="sxs-lookup"><span data-stu-id="7df5d-135">Connections are released back into the pool when they are closed or disposed.</span></span>  
  
### <a name="connection-removal"></a><span data-ttu-id="7df5d-136">移除連接</span><span class="sxs-lookup"><span data-stu-id="7df5d-136">Connection Removal</span></span>

 <span data-ttu-id="7df5d-142">如果連接是與已消失的伺服器連接，且連接共用器尚未偵測到已嚴重損毀的連接並將其標記為無效，則會從集區中描繪連接。</span><span class="sxs-lookup"><span data-stu-id="7df5d-142">If a connection exists to a server that has disappeared, this connection can be drawn from the pool if the connection pooler has not detected the severed connection and marked it as invalid.</span></span> <span data-ttu-id="7df5d-143">此類情況發生時，會產生例外狀況。</span><span class="sxs-lookup"><span data-stu-id="7df5d-143">When this occurs, an exception is generated.</span></span> <span data-ttu-id="7df5d-144">然而，您仍需關閉連接，以將它釋放回集區。</span><span class="sxs-lookup"><span data-stu-id="7df5d-144">However, you must still close the connection in order to release it back into the pool.</span></span>  
  
 <span data-ttu-id="7df5d-145">請不要在您類別之 `Close` 方法中的 `Dispose`、`Connection` 或任何其他 Managed 物件上呼叫 `DataReader` 或 `Finalize`。</span><span class="sxs-lookup"><span data-stu-id="7df5d-145">Do not call `Close` or `Dispose` on a `Connection`, a `DataReader`, or any other managed object in the `Finalize` method of your class.</span></span> <span data-ttu-id="7df5d-146">在完成項中，只需釋放類別直接擁有的 Unmanaged 資源。</span><span class="sxs-lookup"><span data-stu-id="7df5d-146">In a finalizer, only release unmanaged resources that your class owns directly.</span></span> <span data-ttu-id="7df5d-147">如果類別未擁有任何 Unmanaged 資源，請不要在類別定義中包含 `Finalize` 方法。</span><span class="sxs-lookup"><span data-stu-id="7df5d-147">If your class does not own any unmanaged resources, do not include a `Finalize` method in your class definition.</span></span> <span data-ttu-id="7df5d-148">如需詳細資訊，請參閱[回收](../../../../docs/standard/garbage-collection/index.md)。</span><span class="sxs-lookup"><span data-stu-id="7df5d-148">For more information, see [Garbage Collection](../../../../docs/standard/garbage-collection/index.md).</span></span>  
  
### <a name="transaction-support"></a><span data-ttu-id="7df5d-149">異動支援</span><span class="sxs-lookup"><span data-stu-id="7df5d-149">Transaction Support</span></span>
