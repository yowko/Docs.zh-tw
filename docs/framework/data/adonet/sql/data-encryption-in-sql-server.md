# <a name="data-encryption-in-sql-server"></a><span data-ttu-id="149a0-102">在 SQL Server 中加密資料</span><span class="sxs-lookup"><span data-stu-id="149a0-102">Data Encryption in SQL Server</span></span>
<span data-ttu-id="149a0-103">SQL Server 提供使用憑證、非對稱金鑰或對稱金鑰來加密及解密資料的功能，</span><span class="sxs-lookup"><span data-stu-id="149a0-103">SQL Server provides functions to encrypt and decrypt data using a certificate, asymmetric key, or symmetric key.</span></span> <span data-ttu-id="149a0-104">而且可在內部的憑證存放區內管理上述所有項目。</span><span class="sxs-lookup"><span data-stu-id="149a0-104">It manages all of these in an internal certificate store.</span></span> <span data-ttu-id="149a0-105">此存放區會使用加密階層，藉由階層中的上層層級來確保下層層級的憑證及金鑰。</span><span class="sxs-lookup"><span data-stu-id="149a0-105">The store uses an encryption hierarchy that secures certificates and keys at one level with the layer above it in the hierarchy.</span></span> <span data-ttu-id="149a0-106">SQL Server 的此功能區也稱為「秘密儲存區」(Secret Storage)。</span><span class="sxs-lookup"><span data-stu-id="149a0-106">This feature area of SQL Server is called Secret Storage.</span></span>  
  
 <span data-ttu-id="149a0-107">由加密功能所支援的最快加密模式是對稱金鑰加密。</span><span class="sxs-lookup"><span data-stu-id="149a0-107">The fastest mode of encryption supported by the encryption functions is symmetric key encryption.</span></span> <span data-ttu-id="149a0-108">此模式適合用來處理大量的資料。</span><span class="sxs-lookup"><span data-stu-id="149a0-108">This mode is suitable for handling large volumes of data.</span></span> <span data-ttu-id="149a0-109">對稱金鑰可以藉由憑證、密碼或其他對稱金鑰進行加密。</span><span class="sxs-lookup"><span data-stu-id="149a0-109">The symmetric keys can be encrypted by certificates, passwords or other symmetric keys.</span></span>  
  
## <a name="keys-and-algorithms"></a><span data-ttu-id="149a0-110">金鑰和演算法</span><span class="sxs-lookup"><span data-stu-id="149a0-110">Keys and Algorithms</span></span>  
 <span data-ttu-id="149a0-111">SQL Server 支援數種對稱金鑰加密演算法，包括 DES、Triple DES、RC2、RC4、128 位元 RC4、DESX、128 位元 AES、192 位元 AES 和 256 位元 AES 等，</span><span class="sxs-lookup"><span data-stu-id="149a0-111">SQL Server supports several symmetric key encryption algorithms, including DES, Triple DES, RC2, RC4, 128-bit RC4, DESX, 128-bit AES, 192-bit AES, and 256-bit AES.</span></span> <span data-ttu-id="149a0-112">這些演算法是使用 Windows Crypto API 來實作。</span><span class="sxs-lookup"><span data-stu-id="149a0-112">The algorithms are implemented using the Windows Crypto API.</span></span>  
  
 <span data-ttu-id="149a0-113">SQL Server 可在資料庫連接的範圍內維護多個開放式的對稱金鑰。</span><span class="sxs-lookup"><span data-stu-id="149a0-113">Within the scope of a database connection, SQL Server can maintain multiple open symmetric keys.</span></span> <span data-ttu-id="149a0-114">您可從存放區擷取開放式金鑰，並將其用於解密資料。</span><span class="sxs-lookup"><span data-stu-id="149a0-114">An open key is retrieved from the store and is available for decrypting data.</span></span> <span data-ttu-id="149a0-115">在資料解密後，就不需指定要使用的對稱金鑰。</span><span class="sxs-lookup"><span data-stu-id="149a0-115">When a piece of data is decrypted, there is no need to specify the symmetric key to use.</span></span> <span data-ttu-id="149a0-116">每個加密的值都包含加密時所使用金鑰的識別碼 (金鑰 GUID)。</span><span class="sxs-lookup"><span data-stu-id="149a0-116">Each encrypted value contains the key identifier (key GUID) of the key used to encrypt it.</span></span> <span data-ttu-id="149a0-117">如果已解密並開啟了正確的金鑰，則引擎會比對加密的位元組資料流與開放式對稱金鑰，</span><span class="sxs-lookup"><span data-stu-id="149a0-117">The engine matches the encrypted byte stream to an open symmetric key, if the correct key has been decrypted and is open.</span></span> <span data-ttu-id="149a0-118">然後再使用此金鑰來執行解密作業並傳回資料。</span><span class="sxs-lookup"><span data-stu-id="149a0-118">This key is then used to perform decryption and return the data.</span></span> <span data-ttu-id="149a0-119">如果沒有開啟正確的金鑰，則會傳回 NULL。</span><span class="sxs-lookup"><span data-stu-id="149a0-119">If the correct key is not open, NULL is returned.</span></span>  
  
 <span data-ttu-id="149a0-120">如需示範如何使用資料庫中的加密資料的範例，請參閱[加密資料行的](/sql/relational-databases/security/encryption/encrypt-a-column-of-data)。</span><span class="sxs-lookup"><span data-stu-id="149a0-120">For an example that shows how to work with encrypted data in a database, see [Encrypt a Column of Data](/sql/relational-databases/security/encryption/encrypt-a-column-of-data).</span></span>
