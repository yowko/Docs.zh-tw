## <a name="windows-container-types"></a><span data-ttu-id="aceee-211">Windows 容器類型</span><span class="sxs-lookup"><span data-stu-id="aceee-211">Windows container types</span></span>

<span data-ttu-id="aceee-212">就像 Linux 容器，Windows Server 容器是使用 Docker 引擎來管理。</span><span class="sxs-lookup"><span data-stu-id="aceee-212">Like Linux containers, Windows Server containers are managed by using Docker Engine.</span></span> <span data-ttu-id="aceee-213">不同于 Linux 容器，Windows 容器包含兩種不同的容器類型，或執行時間-Windows Server 容器和 Hyper-v 隔離。</span><span class="sxs-lookup"><span data-stu-id="aceee-213">Unlike Linux containers, Windows containers include two different container types, or run times-Windows Server containers and Hyper-V isolation.</span></span>

<span data-ttu-id="aceee-214">**Windows Server 容器**：透過進程和命名空間隔離技術，提供應用程式隔離。</span><span class="sxs-lookup"><span data-stu-id="aceee-214">**Windows Server containers**: Provides application isolation through process and namespace isolation technology.</span></span> <span data-ttu-id="aceee-215">Windows Server 容器與容器主機和所有在主機上執行的容器共用核心。</span><span class="sxs-lookup"><span data-stu-id="aceee-215">A Windows Server container shares a kernel with the container host and all containers that are running on the host.</span></span> <span data-ttu-id="aceee-216">這些容器不會提供惡意的安全性界限，而且不應該用來隔離不受信任的程式碼。</span><span class="sxs-lookup"><span data-stu-id="aceee-216">These containers do not provide a hostile security boundary and should not be used to isolate untrusted code.</span></span> <span data-ttu-id="aceee-217">由於共用核心空間的不同，這些容器需要相同的核心版本和設定。</span><span class="sxs-lookup"><span data-stu-id="aceee-217">Because of the shared kernel space, these containers require the same kernel version and configuration.</span></span>

<span data-ttu-id="aceee-218">**Hyper-v 隔離**：藉由在高度優化的 VM 上執行每個容器，擴充 Windows Server 容器所提供的隔離。</span><span class="sxs-lookup"><span data-stu-id="aceee-218">**Hyper-V isolation**: Expands on the isolation provided by Windows Server Containers by running each container on a highly optimized VM.</span></span> <span data-ttu-id="aceee-219">在此設定中，容器主機的核心不會與相同主機上的其他容器共用。</span><span class="sxs-lookup"><span data-stu-id="aceee-219">In this configuration, the kernel of the container host is not shared with other containers on the same host.</span></span> <span data-ttu-id="aceee-220">這些容器是針對惡意多租使用者裝載所設計，具有與 VM 相同的安全性保證。</span><span class="sxs-lookup"><span data-stu-id="aceee-220">These containers are designed for hostile multitenant hosting, with the same security assurances of a VM.</span></span> <span data-ttu-id="aceee-221">因為這些容器不會與主機或主機上的其他容器共用核心，所以可以使用不同的版本和設定（支援的版本）來執行核心。</span><span class="sxs-lookup"><span data-stu-id="aceee-221">Because these containers don't share the kernel with the host or other containers on the host, they can run kernels with different versions and configurations (with supported versions).</span></span> <span data-ttu-id="aceee-222">例如，Windows 10 上的所有 Windows 容器都使用 Hyper-v 隔離來利用 Windows Server 核心版本和設定。</span><span class="sxs-lookup"><span data-stu-id="aceee-222">For example, all Windows containers on Windows 10 use Hyper-V isolation to utilize the Windows Server kernel version and configuration.</span></span>

<span data-ttu-id="aceee-223">在具有或不含 Hyper-v 隔離的 Windows 上執行容器是一種執行時間決策。</span><span class="sxs-lookup"><span data-stu-id="aceee-223">Running a container on Windows with or without Hyper-V isolation is a run-time decision.</span></span> <span data-ttu-id="aceee-224">一開始，您可以選擇建立具有 Hyper-v 隔離的容器，並在執行時間選擇改為執行 Windows Server 容器。</span><span class="sxs-lookup"><span data-stu-id="aceee-224">You might choose to create the container with Hyper-V isolation initially, and at run time, choose to run it as a Windows Server container instead.</span></span>
