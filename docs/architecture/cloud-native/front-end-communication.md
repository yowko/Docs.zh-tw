# <a name="front-end-client-communication"></a><span data-ttu-id="16185-103">前端用戶端通訊</span><span class="sxs-lookup"><span data-stu-id="16185-103">Front-end client communication</span></span>

<span data-ttu-id="16185-104">在雲端原生系統中，前端用戶端（行動裝置、web 和桌面應用程式）需要通道，才能與獨立的後端微服務互動。</span><span class="sxs-lookup"><span data-stu-id="16185-104">In a cloud-native system, front-end clients (mobile, web, and desktop applications) require a communication channel to interact with independent back-end microservices.</span></span>

<span data-ttu-id="16185-110">使用此方法時，每個微服務都有一個可由前端用戶端存取的公用端點。</span><span class="sxs-lookup"><span data-stu-id="16185-110">With this approach, each microservice has a public endpoint that is accessible by front-end clients.</span></span> <span data-ttu-id="16185-111">在生產環境中，您會將負載平衡器放在微服務的前方，以按比例路由傳送流量。</span><span class="sxs-lookup"><span data-stu-id="16185-111">In a production environment, you'd place a load balancer in front of the microservices, routing traffic proportionately.</span></span>

<span data-ttu-id="16185-112">雖然簡單的執行，但只有簡單的微服務應用程式可接受直接用戶端通訊。</span><span class="sxs-lookup"><span data-stu-id="16185-112">While simple to implement, direct client communication would be acceptable only for simple microservice applications.</span></span> <span data-ttu-id="16185-113">此模式會將前端用戶端與核心後端服務緊密結合，針對許多問題開啟門，包括：</span><span class="sxs-lookup"><span data-stu-id="16185-113">This pattern tightly couples front-end clients to core back-end services, opening the door for a number of problems, including:</span></span>

- <span data-ttu-id="16185-114">用戶端對後端服務重構的敏感程度。</span><span class="sxs-lookup"><span data-stu-id="16185-114">Client susceptibility to back-end service refactoring.</span></span>
- <span data-ttu-id="16185-115">隨著核心後端服務直接公開，會有更大的受攻擊面。</span><span class="sxs-lookup"><span data-stu-id="16185-115">A wider attack surface as core back-end services are directly exposed.</span></span>
- <span data-ttu-id="16185-116">每個微服務的跨領域考慮重複。</span><span class="sxs-lookup"><span data-stu-id="16185-116">Duplication of cross-cutting concerns across each microservice.</span></span>
- <span data-ttu-id="16185-117">過於複雜的用戶端程式代碼-用戶端必須追蹤多個端點，並以彈性的方式處理失敗。</span><span class="sxs-lookup"><span data-stu-id="16185-117">Overly complex client code - clients must keep track of multiple endpoints and handle failures in a resilient way.</span></span>

<span data-ttu-id="16185-118">相反地，廣為接受的雲端設計模式是在前端應用程式與後端服務之間執行[API 閘道服務](../microservices/architect-microservice-container-applications/direct-client-to-microservice-communication-versus-the-api-gateway-pattern.md)。</span><span class="sxs-lookup"><span data-stu-id="16185-118">Instead, a widely accepted cloud design pattern is to implement an [API Gateway Service](../microservices/architect-microservice-container-applications/direct-client-to-microservice-communication-versus-the-api-gateway-pattern.md) between the front-end applications and back-end services.</span></span> <span data-ttu-id="16185-119">模式如圖4-3 所示。</span><span class="sxs-lookup"><span data-stu-id="16185-119">The pattern is shown in Figure 4-3.</span></span>
