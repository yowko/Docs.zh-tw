## <a name="principle-of-least-privilege"></a><span data-ttu-id="8b9da-130">最低許可權原則</span><span class="sxs-lookup"><span data-stu-id="8b9da-130">Principle of least privilege</span></span>

<span data-ttu-id="8b9da-131">電腦安全性性的一項初步概念是最低許可權（POLP）的原則。</span><span class="sxs-lookup"><span data-stu-id="8b9da-131">One of the founding ideas in computer security is the Principle of Least Privilege (POLP).</span></span> <span data-ttu-id="8b9da-132">它實際上是任何形式安全性的基本概念，也就是數位或實體。</span><span class="sxs-lookup"><span data-stu-id="8b9da-132">It's actually a foundational idea in most any form of security be it digital or physical.</span></span> <span data-ttu-id="8b9da-133">簡而言之，原則是任何使用者或程式都應該擁有最少的許可權來執行其工作。</span><span class="sxs-lookup"><span data-stu-id="8b9da-133">In short, the principle is that any user or process should have the smallest number of rights possible to execute its task.</span></span> 

<span data-ttu-id="8b9da-134">例如，假設有一個銀行的出納員：存取 safe 是不尋常的活動。</span><span class="sxs-lookup"><span data-stu-id="8b9da-134">As an example, think of the tellers at a bank: accessing the safe is an uncommon activity.</span></span> <span data-ttu-id="8b9da-135">因此，平均取款無法自行開啟安全。</span><span class="sxs-lookup"><span data-stu-id="8b9da-135">So, the average teller can't open the safe themselves.</span></span> <span data-ttu-id="8b9da-136">若要取得存取權，他們必須透過負責執行額外安全性檢查的銀行經理來提升其要求。</span><span class="sxs-lookup"><span data-stu-id="8b9da-136">To gain access, they need to escalate their request through a bank manager, who performs additional security checks.</span></span> 

<span data-ttu-id="8b9da-137">在電腦系統中，很棒的例子就是連接到資料庫的使用者權限。</span><span class="sxs-lookup"><span data-stu-id="8b9da-137">In a computer system, a fantastic example is the rights of a user connecting to a database.</span></span> <span data-ttu-id="8b9da-138">在許多情況下，都有一個用來建立資料庫結構和執行應用程式的單一使用者帳戶。</span><span class="sxs-lookup"><span data-stu-id="8b9da-138">In many cases, there's a single user account used to both build the database structure and run the application.</span></span> <span data-ttu-id="8b9da-139">除了在極端情況下，執行應用程式的帳戶不需要更新架構資訊的能力。</span><span class="sxs-lookup"><span data-stu-id="8b9da-139">Except in extreme cases, the account running the application doesn't need the ability to update schema information.</span></span> <span data-ttu-id="8b9da-140">應該有數個帳戶可提供不同層級的許可權。</span><span class="sxs-lookup"><span data-stu-id="8b9da-140">There should be several accounts that provide different levels of privilege.</span></span> <span data-ttu-id="8b9da-141">應用程式應該只使用授與讀取和寫入權限給資料表中之資料的許可權層級。</span><span class="sxs-lookup"><span data-stu-id="8b9da-141">The application should only use the permission level that grants read and write access to the data in the tables.</span></span> <span data-ttu-id="8b9da-142">這種保護措施可以排除目標為卸載資料庫資料表或引進惡意觸發程式的攻擊。</span><span class="sxs-lookup"><span data-stu-id="8b9da-142">This kind of protection would eliminate attacks that aimed to drop database tables or introduce malicious triggers.</span></span> 

<span data-ttu-id="8b9da-143">幾乎每個建立雲端原生應用程式的部分都可以因記住最低許可權的原則而受益。</span><span class="sxs-lookup"><span data-stu-id="8b9da-143">Almost every part of building a cloud-native application can benefit from remembering the principle of least privilege.</span></span> <span data-ttu-id="8b9da-144">在角色型存取控制（RBAC）中設定防火牆、網路安全性群組、角色和範圍時，您可以在播放時找到它。</span><span class="sxs-lookup"><span data-stu-id="8b9da-144">You can find it at play when setting up firewalls, network security groups, roles, and scopes in Role-based access control (RBAC).</span></span>
