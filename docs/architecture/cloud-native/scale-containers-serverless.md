## <a name="scaling-out-cloud-native-apps"></a><span data-ttu-id="b5132-119">相應放大雲端原生應用程式</span><span class="sxs-lookup"><span data-stu-id="b5132-119">Scaling out cloud-native apps</span></span>

<span data-ttu-id="b5132-120">雲端原生應用程式可將其他節點或 pod 新增至服務要求，以支援相應放大。</span><span class="sxs-lookup"><span data-stu-id="b5132-120">Cloud-native apps support scaling out by adding additional nodes or pods to service requests.</span></span> <span data-ttu-id="b5132-121">這可以藉由調整應用程式的設定（例如，調整[節點集](https://docs.microsoft.com/azure/aks/use-multiple-node-pools#scale-a-node-pool-manually)區）*或透過自動*調整來手動完成。</span><span class="sxs-lookup"><span data-stu-id="b5132-121">This can be accomplished manually by adjusting configuration settings for the app (for example, [scaling a node pool](https://docs.microsoft.com/azure/aks/use-multiple-node-pools#scale-a-node-pool-manually)), or through *autoscaling*.</span></span> <span data-ttu-id="b5132-122">自動調整會調整應用程式所使用的資源，以回應需求，類似于控溫器如何藉由撥打額外的加熱或冷卻來回應溫度。</span><span class="sxs-lookup"><span data-stu-id="b5132-122">Autoscaling adjusts the resources used by an app in order to respond to demand, similar to how a thermostat responds to temperature by calling for additional heating or cooling.</span></span> <span data-ttu-id="b5132-123">使用自動調整時，會停用手動縮放。</span><span class="sxs-lookup"><span data-stu-id="b5132-123">When using autoscaling, manual scaling is disabled.</span></span>

<span data-ttu-id="b5132-124">AKS 叢集可以透過下列兩種方式的其中一種進行調整：</span><span class="sxs-lookup"><span data-stu-id="b5132-124">AKS clusters can scale in one of two ways:</span></span>
