## <a name="the-cloud"></a><span data-ttu-id="bcf89-143">雲端 。</span><span class="sxs-lookup"><span data-stu-id="bcf89-143">The cloud…</span></span>

<span data-ttu-id="bcf89-144">雲端原生系統會充分利用雲端服務模型。</span><span class="sxs-lookup"><span data-stu-id="bcf89-144">Cloud-native systems take full advantage of the cloud service model.</span></span>

<span data-ttu-id="bcf89-145">這些系統是設計成在動態、虛擬化的雲端環境中，可廣泛使用[平臺即服務（PaaS）](https://azure.microsoft.com/overview/what-is-paas/)計算基礎結構和受控服務。</span><span class="sxs-lookup"><span data-stu-id="bcf89-145">Designed to thrive in a dynamic, virtualized cloud environment, these systems make extensive use of [Platform as a Service (PaaS)](https://azure.microsoft.com/overview/what-is-paas/) compute infrastructure and managed services.</span></span> <span data-ttu-id="bcf89-146">它們會在幾分鐘內將基礎結構視為可*處置*布建，並在需要時調整大小、縮放、移動或損毀–透過自動化。</span><span class="sxs-lookup"><span data-stu-id="bcf89-146">They treat the underlying infrastructure as *disposable* - provisioned in minutes and resized, scaled, moved, or destroyed on demand – via automation.</span></span>

<span data-ttu-id="bcf89-147">考慮到廣為接受的[寵物 DevOps 概念與牛群完全轉換](https://medium.com/@Joachim8675309/devops-concepts-pets-vs-cattle-2380b5aab313)。</span><span class="sxs-lookup"><span data-stu-id="bcf89-147">Consider the widely accepted DevOps concept of [Pets vs. Cattle](https://medium.com/@Joachim8675309/devops-concepts-pets-vs-cattle-2380b5aab313).</span></span> <span data-ttu-id="bcf89-148">在傳統的資料中心，伺服器會被視為*寵物*：實體機器、指定有意義的名稱，以及的關心。</span><span class="sxs-lookup"><span data-stu-id="bcf89-148">In a traditional data center, servers are treated as *Pets*: a physical machine, given a meaningful name, and cared for.</span></span> <span data-ttu-id="bcf89-149">您可以將更多資源新增至同一部電腦（相應增加）來調整規模。</span><span class="sxs-lookup"><span data-stu-id="bcf89-149">You scale by adding more resources to the same machine (scaling up).</span></span> <span data-ttu-id="bcf89-150">如果伺服器變成病假，您會將它返回健康狀態。</span><span class="sxs-lookup"><span data-stu-id="bcf89-150">If the server becomes sick, you nurse it back to health.</span></span> <span data-ttu-id="bcf89-151">萬一伺服器變得無法使用，每個人都會注意到。</span><span class="sxs-lookup"><span data-stu-id="bcf89-151">Should the server become unavailable, everyone notices.</span></span>

<span data-ttu-id="bcf89-152">*牛群完全轉換*服務模型不同。</span><span class="sxs-lookup"><span data-stu-id="bcf89-152">The *Cattle* service model is different.</span></span> <span data-ttu-id="bcf89-153">您會將每個實例布建為虛擬機器或容器。</span><span class="sxs-lookup"><span data-stu-id="bcf89-153">You provision each instance as a virtual machine or container.</span></span> <span data-ttu-id="bcf89-154">它們是相同的，而且指派了系統識別碼，例如，服務-01、Service-02 等等。</span><span class="sxs-lookup"><span data-stu-id="bcf89-154">They're identical and assigned a system identifier such as Service-01, Service-02, and so on.</span></span> <span data-ttu-id="bcf89-155">您可以藉由建立更多專案來進行調整（向外延展）。</span><span class="sxs-lookup"><span data-stu-id="bcf89-155">You scale by creating more of them (scaling out).</span></span> <span data-ttu-id="bcf89-156">當其中一個變成無法使用時，沒有人會注意到。</span><span class="sxs-lookup"><span data-stu-id="bcf89-156">When one becomes unavailable, nobody notices.</span></span>

<span data-ttu-id="bcf89-157">牛群完全轉換模型會涵蓋固定的*基礎結構*。</span><span class="sxs-lookup"><span data-stu-id="bcf89-157">The cattle model embraces *immutable infrastructure*.</span></span> <span data-ttu-id="bcf89-158">伺服器不會進行修復或修改。</span><span class="sxs-lookup"><span data-stu-id="bcf89-158">Servers aren't repaired or modified.</span></span> <span data-ttu-id="bcf89-159">如果其中一個失敗或需要更新，它會被終結並布建一個新的–全都透過自動化完成。</span><span class="sxs-lookup"><span data-stu-id="bcf89-159">If one fails or requires updating, it's destroyed and a new one is provisioned – all done via automation.</span></span>

<span data-ttu-id="bcf89-160">雲端原生系統採用牛群完全轉換服務模型。</span><span class="sxs-lookup"><span data-stu-id="bcf89-160">Cloud-native systems embrace the Cattle service model.</span></span> <span data-ttu-id="bcf89-161">它們會繼續執行，因為基礎結構會相應縮小或相應放大，而不考慮其執行所在的電腦。</span><span class="sxs-lookup"><span data-stu-id="bcf89-161">They continue to run as the infrastructure scales in or out with no regard to the machines upon which they're running.</span></span>

<span data-ttu-id="bcf89-162">Azure 雲端平臺支援這種高度彈性的基礎結構，其中包含自動調整、自我修復和監視功能。</span><span class="sxs-lookup"><span data-stu-id="bcf89-162">The Azure cloud platform supports this type of highly elastic infrastructure with automatic scaling, self-healing, and monitoring capabilities.</span></span>
