## <a name="grpc"></a><span data-ttu-id="9ee03-109">gRPC</span><span class="sxs-lookup"><span data-stu-id="9ee03-109">gRPC</span></span>

<span data-ttu-id="9ee03-110">gRPC 是源自 Google 的開放原始碼通訊。</span><span class="sxs-lookup"><span data-stu-id="9ee03-110">gRPC is an open-source communication that originates from Google.</span></span> <span data-ttu-id="9ee03-111">它是以[遠端程序呼叫（RPC）](https://en.wikipedia.org/wiki/Remote_procedure_call)模型為基礎，在分散式運算中很熱門。</span><span class="sxs-lookup"><span data-stu-id="9ee03-111">It's built upon the [remote procedure call (RPC)](https://en.wikipedia.org/wiki/Remote_procedure_call) model, popular in distributed computing.</span></span> <span data-ttu-id="9ee03-112">在此模型之後，本機用戶端程式會公開同進程方法來執行作業。</span><span class="sxs-lookup"><span data-stu-id="9ee03-112">Following this model, a local client program exposes an in-process method to execute an operation.</span></span> <span data-ttu-id="9ee03-113">在幕後，會在跨分散式網路的跨進程微服務上叫用該呼叫。</span><span class="sxs-lookup"><span data-stu-id="9ee03-113">Behind the scenes, that call is invoked on an out-of-process microservice across a distributed network.</span></span> <span data-ttu-id="9ee03-114">開發人員將作業編碼為本機程序呼叫。</span><span class="sxs-lookup"><span data-stu-id="9ee03-114">The developer codes the operation as a local procedure call.</span></span> <span data-ttu-id="9ee03-115">基礎平臺會將點對點網路通訊、序列化和執行抽象化。</span><span class="sxs-lookup"><span data-stu-id="9ee03-115">The underlying platform abstracts the point-to-point networking communication, serialization, and execution.</span></span>

<span data-ttu-id="9ee03-116">gRPC 是現代化的 RPC 架構，它既輕量又高效能。</span><span class="sxs-lookup"><span data-stu-id="9ee03-116">gRPC is a modern RPC framework that is lightweight and highly performant.</span></span> <span data-ttu-id="9ee03-117">它會使用 HTTP/2 作為其傳輸通訊協定。</span><span class="sxs-lookup"><span data-stu-id="9ee03-117">It uses HTTP/2 for its transport protocol.</span></span> <span data-ttu-id="9ee03-118">與 HTTP 1.1 相容，HTTP/2 具備許多先進的功能：</span><span class="sxs-lookup"><span data-stu-id="9ee03-118">While compatible with HTTP 1.1, HTTP/2 features many advanced capabilities:</span></span>

- <span data-ttu-id="9ee03-119">HTTP 1.1 以純文字傳送資料時，HTTP/2 是二進位通訊協定。</span><span class="sxs-lookup"><span data-stu-id="9ee03-119">While HTTP 1.1 sends data as clear text, HTTP/2 is a binary protocol.</span></span> <span data-ttu-id="9ee03-120">它會使用較少的記憶體，更快地剖析資料、降低網路延遲與速度的相關改善，以及更有效率地管理網路資源。</span><span class="sxs-lookup"><span data-stu-id="9ee03-120">It parses data faster using less memory, reduces network latency with the related improvements to speed, and manages network resources more efficiently.</span></span>
- <span data-ttu-id="9ee03-121">雖然 HTTP 1.1 限制為一次處理一個往返的要求/回應，但 HTTP/2 支援多工傳送，或透過相同連線的多個平行要求。</span><span class="sxs-lookup"><span data-stu-id="9ee03-121">While HTTP 1.1 is limited to processing one round-trip request/response at a time, HTTP /2 supports multiplexing, or multiple parallel requests over the same connection.</span></span>
- <span data-ttu-id="9ee03-122">HTTP/2 支援全雙工或雙向通訊，其中用戶端和伺服器都可以同時進行通訊。</span><span class="sxs-lookup"><span data-stu-id="9ee03-122">HTTP/2 supports full-duplex, or bidirectional communication, where both client and server and can communicate at the same time.</span></span> <span data-ttu-id="9ee03-123">用戶端可以在伺服器送迴響應資料時，同時上傳要求資料。</span><span class="sxs-lookup"><span data-stu-id="9ee03-123">The client can be uploading request data at the same time the server is sending back response data.</span></span>
- <span data-ttu-id="9ee03-124">串流內建于 HTTP/2，這表示要求和回應都可以非同步地串流處理大型資料集。</span><span class="sxs-lookup"><span data-stu-id="9ee03-124">Streaming is built into HTTP/2 meaning that both requests and responses can asynchronously stream large data sets.</span></span>
- <span data-ttu-id="9ee03-125">結合 gRPC 和 HTTP/2，效能會大幅增加。</span><span class="sxs-lookup"><span data-stu-id="9ee03-125">Combining gRPC and HTTP/2, performance dramatically increases.</span></span> <span data-ttu-id="9ee03-126">在[Windows Communication Foundation （WCF）](https://docs.microsoft.com/dotnet/framework/wcf/whats-wcf)用語中中，gPRC 效能符合並超過[NetTCP](https://docs.microsoft.com/dotnet/api/system.servicemodel.nettcpbinding?view=netframework-4.8)系結的速度和效率。</span><span class="sxs-lookup"><span data-stu-id="9ee03-126">In [Windows Communication Foundation (WCF)](https://docs.microsoft.com/dotnet/framework/wcf/whats-wcf) parlance, gPRC performance meets and exceeds the speed and efficiency of [NetTCP bindings](https://docs.microsoft.com/dotnet/api/system.servicemodel.nettcpbinding?view=netframework-4.8).</span></span> <span data-ttu-id="9ee03-127">不過，不同于 NetTCP，gRPC 不受限於 Microsoft 語言， C#例如或 VB.NET。</span><span class="sxs-lookup"><span data-stu-id="9ee03-127">However, unlike NetTCP, gRPC isn't constrained to Microsoft languages such as C# or VB.NET.</span></span>

<span data-ttu-id="9ee03-128">跨最受歡迎的平臺（包括 JAVA、 C#、Golang 和 NodeJS）都支援 gRPC。</span><span class="sxs-lookup"><span data-stu-id="9ee03-128">gRPC is supported across most popular platforms, including Java, C#, Golang and NodeJS.</span></span> 

## <a name="protocol-buffers"></a><span data-ttu-id="9ee03-129">通訊協定緩衝區</span><span class="sxs-lookup"><span data-stu-id="9ee03-129">Protocol Buffers</span></span>

<span data-ttu-id="9ee03-130">gRPC 採用另一種稱為[通訊協定緩衝區](https://developers.google.com/protocol-buffers/docs/overview)或 Protobuf 訊息的開放原始碼技術，來傳送和接收資料。</span><span class="sxs-lookup"><span data-stu-id="9ee03-130">gRPC embraces another open-source technology called [Protocol Buffers](https://developers.google.com/protocol-buffers/docs/overview) or Protobuf messages to send and receive data.</span></span> <span data-ttu-id="9ee03-131">類似于[WCF 資料合約](https://docs.microsoft.com/dotnet/framework/wcf/feature-details/using-data-contracts)，Protobuf 會將系統的結構化資料序列化，以便進行讀取和寫入。</span><span class="sxs-lookup"><span data-stu-id="9ee03-131">Similar to a [WCF Data Contract](https://docs.microsoft.com/dotnet/framework/wcf/feature-details/using-data-contracts), Protobuf serializes structured data for systems to read and write.</span></span> <span data-ttu-id="9ee03-132">它可降低人類看得懂的格式（例如 XML 或 JSON）所產生的額外負荷。</span><span class="sxs-lookup"><span data-stu-id="9ee03-132">It reduces the overhead that human-readable formats like XML or JSON incur.</span></span>

<span data-ttu-id="9ee03-133">許多物件序列化技術在執行時間會反映在資料物件的結構上。</span><span class="sxs-lookup"><span data-stu-id="9ee03-133">Many object serialization techniques reflect across the structure of data objects at run-time.</span></span> <span data-ttu-id="9ee03-134">Protobuf 需要您預先定義具有平臺中立語言（通訊協定緩衝區語言）的結構。</span><span class="sxs-lookup"><span data-stu-id="9ee03-134">Protobuf requires you to define the structure up front with a platform-agnostic language (Protocol Buffer Language).</span></span> <span data-ttu-id="9ee03-135">每個定義都會儲存在「. proto」檔案中。</span><span class="sxs-lookup"><span data-stu-id="9ee03-135">Each definition is stored in a ".proto" file.</span></span> <span data-ttu-id="9ee03-136">然後使用 Protobuf 編譯器「Proton」，為任何支援的平臺產生用戶端和伺服器程式碼。</span><span class="sxs-lookup"><span data-stu-id="9ee03-136">Then using Protobuf compiler, "Proton," you generate client and server code for any of the supported platforms.</span></span> <span data-ttu-id="9ee03-137">產生的程式碼已針對資料的快速序列化/還原序列化進行優化。</span><span class="sxs-lookup"><span data-stu-id="9ee03-137">The generated code is optimized for fast serialization/deserialization of data.</span></span> <span data-ttu-id="9ee03-138">在執行時間，每個訊息都會包裝在強型別服務合約中，並以標準的 Protobuf 標記法進行序列化。</span><span class="sxs-lookup"><span data-stu-id="9ee03-138">At runtime, each message is wrapped in the strongly-typed service contract and serialized in a standard Protobuf representation.</span></span>
