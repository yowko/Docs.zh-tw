## <a name="blazor-server-apps"></a><span data-ttu-id="e305c-133">Blazor 伺服器應用程式</span><span class="sxs-lookup"><span data-stu-id="e305c-133">Blazor Server apps</span></span>

<span data-ttu-id="e305c-134">回想一下[Blazor 架構](architecture-comparison.md#blazor)討論，Blazor 元件會將其輸出轉譯為名為的`RenderTree`中繼抽象概念。</span><span class="sxs-lookup"><span data-stu-id="e305c-134">Recall from the [Blazor architecture](architecture-comparison.md#blazor) discussion that Blazor components render their output to an intermediate abstraction called a `RenderTree`.</span></span> <span data-ttu-id="e305c-135">然後，Blazor 架構會比較呈現的內容與先前轉譯的內容。</span><span class="sxs-lookup"><span data-stu-id="e305c-135">The Blazor framework then compares what was rendered with what was previously rendered.</span></span> <span data-ttu-id="e305c-136">這些差異會套用至 DOM。</span><span class="sxs-lookup"><span data-stu-id="e305c-136">The differences are applied to the DOM.</span></span> <span data-ttu-id="e305c-137">Blazor 元件會與其呈現輸出的套用方式分離。</span><span class="sxs-lookup"><span data-stu-id="e305c-137">Blazor components are decoupled from how their rendered output is applied.</span></span> <span data-ttu-id="e305c-138">因此，元件本身不需要與更新 UI 的進程在相同的進程中執行。</span><span class="sxs-lookup"><span data-stu-id="e305c-138">Consequently, the components themselves don't have to run in the same process as the process updating the UI.</span></span> <span data-ttu-id="e305c-139">事實上，它們甚至不需要在同一部電腦上執行。</span><span class="sxs-lookup"><span data-stu-id="e305c-139">In fact, they don't even have to run on the same machine.</span></span>

<span data-ttu-id="e305c-140">在 Blazor 伺服器應用程式中，元件會在伺服器上執行，而不是在瀏覽器中的用戶端上執行。</span><span class="sxs-lookup"><span data-stu-id="e305c-140">In Blazor Server apps, the components run on the server instead of client-side in the browser.</span></span> <span data-ttu-id="e305c-141">在瀏覽器中發生的 UI 事件會透過即時連線傳送至伺服器。</span><span class="sxs-lookup"><span data-stu-id="e305c-141">UI events that occur in the browser are sent to the server over a real-time connection.</span></span> <span data-ttu-id="e305c-142">事件會分派至正確的元件實例。</span><span class="sxs-lookup"><span data-stu-id="e305c-142">The events are dispatched to the correct component instances.</span></span> <span data-ttu-id="e305c-143">元件會轉譯，而匯出的 UI diff 會序列化並傳送至它套用至 DOM 的瀏覽器。</span><span class="sxs-lookup"><span data-stu-id="e305c-143">The components render, and the calculated UI diff is serialized and sent to the browser where it's applied to the DOM.</span></span>

<span data-ttu-id="e305c-145">如果您已使用 ASP.NET AJAX 和<xref:System.Web.UI.UpdatePanel>控制項，Blazor 伺服器裝載模型可能會很熟悉。</span><span class="sxs-lookup"><span data-stu-id="e305c-145">The Blazor Server hosting model may sound familiar if you've used ASP.NET AJAX and the <xref:System.Web.UI.UpdatePanel> control.</span></span> <span data-ttu-id="e305c-146">`UpdatePanel`控制項會處理部分頁面更新的套用，以回應頁面上的觸發程式事件。</span><span class="sxs-lookup"><span data-stu-id="e305c-146">The `UpdatePanel` control handles applying partial page updates in response to trigger events on the page.</span></span> <span data-ttu-id="e305c-147">觸發時， `UpdatePanel`會要求部分更新，然後套用它，而不需要重新整理頁面。</span><span class="sxs-lookup"><span data-stu-id="e305c-147">When triggered, the `UpdatePanel` requests a partial update and then applies it without needing to refresh the page.</span></span> <span data-ttu-id="e305c-148">UI 的狀態是使用`ViewState`來管理。</span><span class="sxs-lookup"><span data-stu-id="e305c-148">The state of the UI is managed using `ViewState`.</span></span> <span data-ttu-id="e305c-149">Blazor 伺服器應用程式稍有不同，因為應用程式需要與用戶端使用中的連接。</span><span class="sxs-lookup"><span data-stu-id="e305c-149">Blazor Server apps are slightly different in that the app requires an active connection with the client.</span></span> <span data-ttu-id="e305c-150">此外，所有 UI 狀態都會保留在伺服器上。</span><span class="sxs-lookup"><span data-stu-id="e305c-150">Additionally, all UI state is maintained on the server.</span></span> <span data-ttu-id="e305c-151">除了這些差異之外，這兩個模型在概念上類似。</span><span class="sxs-lookup"><span data-stu-id="e305c-151">Aside from those differences, the two models are conceptually similar.</span></span>
