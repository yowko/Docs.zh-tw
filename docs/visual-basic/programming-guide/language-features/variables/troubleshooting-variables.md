## <a name="other-components-cannot-access-your-variable"></a><span data-ttu-id="17928-119">其他元件無法存取您的變數</span><span class="sxs-lookup"><span data-stu-id="17928-119">Other Components Cannot Access Your Variable</span></span>  
 <span data-ttu-id="17928-120">Visual Basic 名稱不*不區分大小寫*。</span><span class="sxs-lookup"><span data-stu-id="17928-120">Visual Basic names are *case-insensitive*.</span></span> <span data-ttu-id="17928-121">如果兩個名稱只有字母大小寫不同，則編譯器會將它們解譯成相同的名稱。</span><span class="sxs-lookup"><span data-stu-id="17928-121">If two names differ in alphabetic case only, the compiler interprets them as the same name.</span></span> <span data-ttu-id="17928-122">例如，它會將 `ABC` 和 `abc` 視為相同的宣告元素。</span><span class="sxs-lookup"><span data-stu-id="17928-122">For example, it considers `ABC` and `abc` to refer to the same declared element.</span></span>  
  
 <span data-ttu-id="17928-123">不過，Common Language Runtime (CLR) 使用 *區分大小寫* 的繫結。</span><span class="sxs-lookup"><span data-stu-id="17928-123">However, the common language runtime (CLR) uses *case-sensitive* binding.</span></span> <span data-ttu-id="17928-124">因此，當您產生一個組件或 DLL 並讓其他組件使用時，您的名稱將會區分大小寫。</span><span class="sxs-lookup"><span data-stu-id="17928-124">Therefore, when you produce an assembly or a DLL and make it available to other assemblies, your names are no longer case-insensitive.</span></span> <span data-ttu-id="17928-125">例如，如果您使用名為 `ABC`的元素來定義類別，而其他組件透過 Common Language Runtime 使用您的類別，則這些組件必須以 `ABC`來表示該元素。</span><span class="sxs-lookup"><span data-stu-id="17928-125">For example, if you define a class with an element called `ABC`, and other assemblies make use of your class through the common language runtime, they must refer to the element as `ABC`.</span></span> <span data-ttu-id="17928-126">如果您隨後重新編譯類別，並將元素的名稱變更為 `abc`，則其他使用此類別的組件就無法再存取該元素。</span><span class="sxs-lookup"><span data-stu-id="17928-126">If you subsequently recompile your class and change the element's name to `abc`, the other assemblies using your class can no longer access that element.</span></span> <span data-ttu-id="17928-127">因此，當您發行組件的更新版本時，不應該變更任何公用元素的字母大小寫。</span><span class="sxs-lookup"><span data-stu-id="17928-127">Therefore, when you release an updated version of an assembly, you should not change the alphabetic case of any public elements.</span></span>
