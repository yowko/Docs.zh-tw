# <a name="class-classname-has-no-accessible-sub-new-and-cannot-be-inherited"></a><span data-ttu-id="3a029-102">類別\<類別名稱 >' 沒有可存取 ' Sub New' 而且無法被繼承</span><span class="sxs-lookup"><span data-stu-id="3a029-102">Class '\<classname>' has no accessible 'Sub New' and cannot be inherited</span></span>
<span data-ttu-id="3a029-103">類別會使用 [Inherits Statement](../../visual-basic/language-reference/statements/inherits-statement.md) 指定基底類別，但它無法存取預期基底類別上的任何建構函式。</span><span class="sxs-lookup"><span data-stu-id="3a029-103">A class uses an [Inherits Statement](../../visual-basic/language-reference/statements/inherits-statement.md) to specify a base class, but it cannot access any constructor on the intended base class.</span></span>  
  
 <span data-ttu-id="3a029-104">如果預期基底類別沒有建構函式，或者它的建構函式具有可防止從另一個類別存取的存取層級，也可能會發生這種情況。</span><span class="sxs-lookup"><span data-stu-id="3a029-104">This can happen if the intended base class has no constructors or if it has constructors with access levels that prevent access from another class.</span></span>  
  
 <span data-ttu-id="3a029-105">當您繼承類別時，建構函式應該呼叫基底類別建構函式使用[MyBase](~/docs/visual-basic/programming-guide/program-structure/me-my-mybase-and-myclass.md#mybase)。</span><span class="sxs-lookup"><span data-stu-id="3a029-105">When you inherit a class, your constructor should call the base class constructor using [MyBase](~/docs/visual-basic/programming-guide/program-structure/me-my-mybase-and-myclass.md#mybase).</span></span> <span data-ttu-id="3a029-106">如果您未進行這個呼叫，或者您甚至未撰寫明確的建構函式，Visual Basic 會產生可呼叫 `MyBase.New()`的隱含建構函式。</span><span class="sxs-lookup"><span data-stu-id="3a029-106">If you do not make this call, or if you do not even write an explicit constructor, Visual Basic generates an implicit constructor that calls `MyBase.New()`.</span></span>  
  
 <span data-ttu-id="3a029-107">**錯誤 ID:** BC31399</span><span class="sxs-lookup"><span data-stu-id="3a029-107">**Error ID:** BC31399</span></span>  
  
## <a name="to-correct-this-error"></a><span data-ttu-id="3a029-108">更正這個錯誤</span><span class="sxs-lookup"><span data-stu-id="3a029-108">To correct this error</span></span>  
  
1. <span data-ttu-id="3a029-109">如果您有預期基底類別的原始檔控制，則請變更其至少一個建構函式的存取層級，讓另一個類別可以存取它們。</span><span class="sxs-lookup"><span data-stu-id="3a029-109">If you have source control over the intended base class, then change the access level of at least one of its constructors so that another class can access them.</span></span>  
  
2. <span data-ttu-id="3a029-110">如果您無法變更預期基底類別建構函式的存取層級，則會從其他類別繼承存取層級，或根本沒有存取層級。</span><span class="sxs-lookup"><span data-stu-id="3a029-110">If you cannot change the access levels of the intended base class constructors, then inherit from a different class or not at all.</span></span>
