 <span data-ttu-id="3ab49-105">嘗試使用類型推斷，來判斷評估泛型程序呼叫時之類型參數的資料類型。</span><span class="sxs-lookup"><span data-stu-id="3ab49-105">An attempt has been made to use type inference to determine the data type or types of the type parameter or parameters when evaluating a call to a generic procedure.</span></span> <span data-ttu-id="3ab49-106">編譯器找不到符合所有引數之條件約束的資料類型。</span><span class="sxs-lookup"><span data-stu-id="3ab49-106">The compiler could not find a data type that meets the constraints of all the arguments.</span></span> <span data-ttu-id="3ab49-107">因此，它已回報這個錯誤。</span><span class="sxs-lookup"><span data-stu-id="3ab49-107">Therefore, it reported this error.</span></span>

>  <span data-ttu-id="3ab49-108">當無法指定引數時 (例如，針對查詢運算式中的查詢運算子)，會顯示不含第二句的錯誤訊息。</span><span class="sxs-lookup"><span data-stu-id="3ab49-108">When specifying arguments is not an option (for example, for query operators in query expressions), the error message appears without the second sentence.</span></span>
