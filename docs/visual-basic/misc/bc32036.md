# <a name="class-classname1-must-declare-a-sub-new-because-its-base-class-classname2-has-more-than-one-accessible-sub-new-that-can-be-called-with-no-arguments"></a>類別 '\<類別名稱 1> > ' 必須宣告 ' sub new ', 因為其基類 '\<類別名稱 2> > ' 有多個可呼叫且不含引數的可存取 ' Sub new '
衍生類別不會宣告一個函式, 而且 Visual Basic 無法產生一個, 因為它無法判斷要呼叫哪個基類的函式。  
  
 當衍生類別未宣告函式時, Visual Basic 會嘗試產生呼叫`MyBase.New()`的隱含無參數函數。 如果無法在沒有引數的情況下呼叫的基類中沒有可存取的函式, 或如果有一個以上的, Visual Basic 無法產生隱含的函數。  
  
 例如，如果一個基底類別建構函式具有單一 `Optional` 引數，而另一個具有單一 `ParamArray` 引數，則可能會發生這種情況。 上述每一個都可以在不使用引數的情況下呼叫。  
  
 **錯誤識別碼:** BC32036  
  
## <a name="to-correct-this-error"></a>更正這個錯誤  
  
1. 在衍生類別的某個位置，宣告和實作至少一個 `Sub New` 建構函式。  
  
2. 加入基底類別建構函式 `MyBase.New()`的呼叫，以作為每個 `Sub New`的第一行。
