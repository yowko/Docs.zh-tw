## EditorConfig

<span data-ttu-id="d8d28-109">[EditorConfig](/visualstudio/ide/create-portable-custom-editor-options) 檔案是用來提供 **適用于特定原始程式檔或資料夾的選項**。</span><span class="sxs-lookup"><span data-stu-id="d8d28-109">[EditorConfig](/visualstudio/ide/create-portable-custom-editor-options) files are used to provide **options that apply to specific source files or folders**.</span></span> <span data-ttu-id="d8d28-110">選項會放置在區段標頭下，以識別適用的檔案和資料夾。</span><span class="sxs-lookup"><span data-stu-id="d8d28-110">Options are placed under section headers to identify the applicable files and folders.</span></span> <span data-ttu-id="d8d28-111">為您要設定的每個規則加入一個專案，並將它放在對應的副檔名區段下，例如 `[*.cs]` 。</span><span class="sxs-lookup"><span data-stu-id="d8d28-111">Add an entry for each rule you want to configure, and place it under the corresponding file extension section, for example, `[*.cs]`.</span></span>

```ini

<option_name> = <option_value>
```

<span data-ttu-id="d8d28-112">在上述範例中， `[*.cs]` 是一個 editorconfig 區段標頭，可選取目前資料夾內副檔名為 file 的所有 c # 檔案 `.cs` ，包括子資料夾。</span><span class="sxs-lookup"><span data-stu-id="d8d28-112">In the above example, `[*.cs]` is an editorconfig section header to select all C# files with `.cs` file extension within the current folder, including subfolders.</span></span> <span data-ttu-id="d8d28-113">後續的專案 `<option_name> = <option_value>` 是分析器選項，將會套用至所有 c # 檔案。</span><span class="sxs-lookup"><span data-stu-id="d8d28-113">The subsequent entry, `<option_name> = <option_value>`, is an analyzer option that will be applied to all the C# files.</span></span>

<span data-ttu-id="d8d28-114">您可以藉 EditorConfig 由將檔案放在對應的目錄中，將檔案慣例套用至資料夾、專案或整個存放庫。</span><span class="sxs-lookup"><span data-stu-id="d8d28-114">You can apply EditorConfig file conventions to a folder, a project, or an entire repo by placing the file in the corresponding directory.</span></span> <span data-ttu-id="d8d28-115">這些選項會在組建階段執行分析時，以及您在 Visual Studio 中編輯程式碼時套用。</span><span class="sxs-lookup"><span data-stu-id="d8d28-115">These options are applied when executing the analysis at build time and while you edit code in Visual Studio.</span></span>

<span data-ttu-id="d8d28-116">如果您有現有的 *editorconfig* 檔案可用於編輯器設定，例如縮排大小或是否要修剪尾端空白字元，則可以將您的程式碼分析設定選項放在相同的檔案中。</span><span class="sxs-lookup"><span data-stu-id="d8d28-116">If you have an existing *.editorconfig* file for editor settings such as indent size or whether to trim trailing whitespace, you can place your code analysis configuration options in the same file.</span></span>

> [!TIP]
> <span data-ttu-id="d8d28-117">Visual Studio 提供 *editorconfig* 專案範本，可讓您輕鬆地將其中一個檔案新增至您的專案。</span><span class="sxs-lookup"><span data-stu-id="d8d28-117">Visual Studio provides an *.editorconfig* item template that makes is easy to add one of these files to your project.</span></span> <span data-ttu-id="d8d28-118">如需詳細資訊，請參閱 [將檔案新增 EditorConfig 至專案](/visualstudio/ide/create-portable-custom-editor-options#add-an-editorconfig-file-to-a-project)。</span><span class="sxs-lookup"><span data-stu-id="d8d28-118">For more information, see [Add an EditorConfig file to a project](/visualstudio/ide/create-portable-custom-editor-options#add-an-editorconfig-file-to-a-project).</span></span>
