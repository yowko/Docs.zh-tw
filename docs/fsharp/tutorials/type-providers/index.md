# <a name="type-providers"></a><span data-ttu-id="d3c8e-103">型別提供者</span><span class="sxs-lookup"><span data-stu-id="d3c8e-103">Type Providers</span></span>

<span data-ttu-id="d3c8e-104">F# 型別提供者是一個元件，該元件會提供類型、屬性和方法讓您在程式中使用。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-104">An F# type provider is a component that provides types, properties, and methods for use in your program.</span></span> <span data-ttu-id="d3c8e-105">型別提供者會產生所謂的「**提供類型**」，這是F#由編譯器所產生，而且是以外部資料源為基礎。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-105">Type Providers generate what are known as **Provided Types**, which are generated by the F# compiler and are based on an external data source.</span></span>

<span data-ttu-id="d3c8e-106">例如，適用于F# SQL 的類型提供者可以產生類型，代表關係資料庫中的資料表和資料行。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-106">For example, an F# Type Provider for SQL can generate types representing tables and columns in a relational database.</span></span> <span data-ttu-id="d3c8e-107">事實上，這就是[SQLProvider](https://fsprojects.github.io/SQLProvider/)型別提供者的功能。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-107">In fact, this is what the [SQLProvider](https://fsprojects.github.io/SQLProvider/) Type Provider does.</span></span>

<span data-ttu-id="d3c8e-108">提供的類型取決於型別提供者的輸入參數。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-108">Provided Types depend on input parameters to a Type Provider.</span></span> <span data-ttu-id="d3c8e-109">這類輸入可以是範例資料來源（例如 JSON 架構檔案）、直接指向外部服務的 URL，或是資料來源的連接字串。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-109">Such input can be a sample data source (such as a JSON schema file), a URL pointing directly to an external service, or a connection string to a data source.</span></span> <span data-ttu-id="d3c8e-110">型別提供者也可以確保類型的群組只視需要展開;也就是說，如果您的程式實際參考型別，它們就會展開。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-110">A Type Provider can also ensure that groups of types are only expanded on demand; that is, they are expanded if the types are actually referenced by your program.</span></span> <span data-ttu-id="d3c8e-111">這樣就能以強類型的方式視需要直接整合大型的資訊空間，例如線上資料市場。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-111">This allows for the direct, on-demand integration of large-scale information spaces such as online data markets in a strongly typed way.</span></span>

## <a name="generative-and-erased-type-providers"></a><span data-ttu-id="d3c8e-112">有生產力和已清除的類型提供者</span><span class="sxs-lookup"><span data-stu-id="d3c8e-112">Generative and Erased Type Providers</span></span>

<span data-ttu-id="d3c8e-113">型別提供者有兩種形式：有生產力和清除。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-113">Type Providers come in two forms: Generative and Erased.</span></span>

<span data-ttu-id="d3c8e-114">有生產力類型提供者會產生可當做 .NET 類型寫入至產生它們之元件中的類型。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-114">Generative Type Providers produce types that can be written as .NET types into the assembly in which they are produced.</span></span> <span data-ttu-id="d3c8e-115">這可讓它們從其他元件中的程式碼使用。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-115">This allows them to be consumed from code in other assemblies.</span></span> <span data-ttu-id="d3c8e-116">這表示，資料來源的具類型標記法通常必須是可以用 .NET 類型來表示的一種。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-116">This means that the typed representation of the data source must generally be one that is feasible to represent with .NET types.</span></span>

<span data-ttu-id="d3c8e-117">清除類型提供者會產生只能在其產生來源的元件或專案中使用的類型。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-117">Erasing Type Providers produce types that can only be consumed in the assembly or project they are generated from.</span></span> <span data-ttu-id="d3c8e-118">這些類型是暫時的;也就是說，它們不會寫入元件中，而且無法由其他元件中的程式碼使用。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-118">The types are ephemeral; that is, they are not written into an assembly and cannot be consumed by code in other assemblies.</span></span> <span data-ttu-id="d3c8e-119">它們可以包含*延遲*的成員，讓您可以從可能的無限資訊空間使用提供的類型。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-119">They can contain *delayed* members, allowing you to use provided types from a potentially infinite information space.</span></span> <span data-ttu-id="d3c8e-120">它們對於使用大型且相互連接之資料來源的小型子集很有用。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-120">They are useful for using a small subset of a large and interconnected data source.</span></span>

## <a name="commonly-used-type-providers"></a><span data-ttu-id="d3c8e-121">常用的類型提供者</span><span class="sxs-lookup"><span data-stu-id="d3c8e-121">Commonly used Type Providers</span></span>

<span data-ttu-id="d3c8e-122">下列廣泛使用的程式庫包含不同用途的類型提供者：</span><span class="sxs-lookup"><span data-stu-id="d3c8e-122">The following widely-used libraries contain Type Providers for different uses:</span></span>

- <span data-ttu-id="d3c8e-123">[Fsharp.core：資料](https://fsharp.github.io/FSharp.Data/)報括 JSON、XML、CSV 和 HTML 檔案格式和資源的類型提供者。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-123">[FSharp.Data](https://fsharp.github.io/FSharp.Data/) includes Type Providers for JSON, XML, CSV, and HTML document formats and resources.</span></span>
- <span data-ttu-id="d3c8e-124">[SQLProvider](https://fsprojects.github.io/SQLProvider/)透過物件對應和針對這些資料來源的 LINQ 查詢， F#提供關聯資料庫的強型別存取。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-124">[SQLProvider](https://fsprojects.github.io/SQLProvider/) provides strongly-typed access to relation databases through object mapping and F# LINQ queries against these data sources.</span></span>
- <span data-ttu-id="d3c8e-125">[SqlClient](https://fsprojects.github.io/FSharp.Data.SqlClient/)具有一組類型提供者，適用于編譯時期已核取的 t-sql F#。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-125">[FSharp.Data.SqlClient](https://fsprojects.github.io/FSharp.Data.SqlClient/) has a set of type providers for compile-time checked embedding of T-SQL in F#.</span></span>
- <span data-ttu-id="d3c8e-126">[Azure 儲存體類型提供者](https://fsprojects.github.io/AzureStorageTypeProvider/)提供適用于 Azure Blob、資料表和佇列的類型，可讓您存取這些資源，而不需要在整個程式中將資源名稱指定為字串。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-126">[Azure Storage Type provider](https://fsprojects.github.io/AzureStorageTypeProvider/) provides types for Azure Blobs, Tables, and Queues, allowing you to access these resources without needing to specify resource names as strings throughout your program.</span></span>
- <span data-ttu-id="d3c8e-127">[GraphQL](https://fsprojects.github.io/FSharp.Data.GraphQL/index.html)包含**GraphQLProvider**，它會根據 URL 所指定的 GraphQL 伺服器提供類型。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-127">[FSharp.Data.GraphQL](https://fsprojects.github.io/FSharp.Data.GraphQL/index.html) contains the **GraphQLProvider**, which provides types based on a GraphQL server specified by URL.</span></span>

<span data-ttu-id="d3c8e-128">必要時，您可以[建立自己的自訂類型提供者](creating-a-type-provider.md)，或已由其他人建立的參考型別提供者。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-128">Where necessary, you can [create your own custom type providers](creating-a-type-provider.md), or reference type providers that have been created by others.</span></span> <span data-ttu-id="d3c8e-129">例如，假設您的組織的資料服務提供大量且不斷增加的具名資料集，各資料集都有自己的穩定資料結構描述。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-129">For example, assume your organization has a data service providing a large and growing number of named data sets, each with its own stable data schema.</span></span> <span data-ttu-id="d3c8e-130">您可能選擇建立一個類型提供者，以強類型方式讀取結構描述並對程式設計人員呈現最新的可用資料集。</span><span class="sxs-lookup"><span data-stu-id="d3c8e-130">You may choose to create a type provider that reads the schemas and presents the latest available data sets to the programmer in a strongly typed way.</span></span>
