<span data-ttu-id="fc8a1-103">C# 語言的例外狀況處理功能可協助您處理在程式執行時發生的任何未預期或例外狀況。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-103">The C# language's exception handling features help you deal with any unexpected or exceptional situations that occur when a program is running.</span></span> <span data-ttu-id="fc8a1-104">例外狀況處理會使用 `try`、`catch` 和 `finally` 關鍵字來嘗試可能失敗的動作，以便在您決定這樣做很合理時處理失敗，之後再清除資源。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-104">Exception handling uses the `try`, `catch`, and `finally` keywords to try actions that may not succeed, to handle failures when you decide that it is reasonable to do so, and to clean up resources afterward.</span></span> <span data-ttu-id="fc8a1-105">例外狀況可以由 Common Language Runtime (CLR)、.NET Framework 或任何協力廠商程式庫，或者應用程式程式碼來產生。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-105">Exceptions can be generated by the common language runtime (CLR), by the .NET Framework or any third-party libraries, or by application code.</span></span> <span data-ttu-id="fc8a1-106">例外狀況是使用 `throw` 關鍵字所建立。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-106">Exceptions are created by using the `throw` keyword.</span></span>  
  
 <span data-ttu-id="fc8a1-107">在許多情況下，例外狀況可能不是由您的程式碼直接呼叫的方法所擲回，而是由呼叫堆疊中更下方的另一個方法所擲回。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-107">In many cases, an exception may be thrown not by a method that your code has called directly, but by another method further down in the call stack.</span></span> <span data-ttu-id="fc8a1-108">發生這種情況時，CLR 將回溯堆疊，針對特定的例外狀況型別尋找具有 `catch` 區塊的方法，而它將會執行第一個這類 `catch` 區塊 (如果找到)。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-108">When this happens, the CLR will unwind the stack, looking for a method with a `catch` block for the specific exception type, and it will execute the first such `catch` block that if finds.</span></span> <span data-ttu-id="fc8a1-109">如果在呼叫堆疊中的任何地方都找不到適當的 `catch` 區塊，它將會終止處理序，並向使用者顯示一則訊息。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-109">If it finds no appropriate `catch` block anywhere in the call stack, it will terminate the process and display a message to the user.</span></span>

## <a name="exceptions-overview"></a><span data-ttu-id="fc8a1-112">例外狀況概觀</span><span class="sxs-lookup"><span data-stu-id="fc8a1-112">Exceptions Overview</span></span>  
 <span data-ttu-id="fc8a1-113">例外狀況具有下列屬性：</span><span class="sxs-lookup"><span data-stu-id="fc8a1-113">Exceptions have the following properties:</span></span>  
  
-   <span data-ttu-id="fc8a1-114">例外狀況是最終全都衍生自 `System.Exception` 的型別。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-114">Exceptions are types that all ultimately derive from `System.Exception`.</span></span>  
  
-   <span data-ttu-id="fc8a1-115">在可能擲回例外狀況的陳述式前後使用 `try` 區塊。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-115">Use a `try` block around the statements that might throw exceptions.</span></span>  
  
-   <span data-ttu-id="fc8a1-116">一旦 `try` 區塊中發生例外狀況之後，控制流程就會跳至第一個相關聯的例外狀況處理常式，此處理常式存在於呼叫堆疊中的任何位置。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-116">Once an exception occurs in the `try` block, the flow of control jumps to the first associated exception handler that is present anywhere in the call stack.</span></span> <span data-ttu-id="fc8a1-117">在 C# 中，`catch` 關鍵字是用來定義例外狀況處理常式。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-117">In C#, the `catch` keyword is used to define an exception handler.</span></span>  
  
-   <span data-ttu-id="fc8a1-118">如果沒有指定例外狀況的例外狀況處理常式存在，程式就會停止執行並出現錯誤訊息。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-118">If no exception handler for a given exception is present, the program stops executing with an error message.</span></span>  
  
-   <span data-ttu-id="fc8a1-119">除非您可以處理它，否則不要攔截例外狀況，並讓應用程式保持已知的狀態。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-119">Do not catch an exception unless you can handle it and leave the application in a known state.</span></span> <span data-ttu-id="fc8a1-120">如果您攔截 `System.Exception`，請在 `catch` 區塊結尾使用 `throw` 關鍵字重新擲回它。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-120">If you catch `System.Exception`, rethrow it using the `throw` keyword at the end of the `catch` block.</span></span>  
  
-   <span data-ttu-id="fc8a1-121">如果 `catch` 區塊定義了例外狀況變數，您可以使用它來取得所發生例外狀況型別的詳細資訊。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-121">If a `catch` block defines an exception variable, you can use it to obtain more information about the type of exception that occurred.</span></span>  
  
-   <span data-ttu-id="fc8a1-122">例外狀況可以透過程式使用 `throw` 關鍵字明確地產生。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-122">Exceptions can be explicitly generated by a program by using the `throw` keyword.</span></span>  
  
-   <span data-ttu-id="fc8a1-123">例外狀況物件包含錯誤的詳細資訊，例如呼叫堆疊的狀態和錯誤狀態的文字描述。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-123">Exception objects contain detailed information about the error, such as the state of the call stack and a text description of the error.</span></span>  
  
-   <span data-ttu-id="fc8a1-124">即使擲回例外狀況，`finally` 區塊中的程式碼也會執行。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-124">Code in a `finally` block is executed even if an exception is thrown.</span></span> <span data-ttu-id="fc8a1-125">使用 `finally` 區塊來釋放資源，例如，關閉已在 `try` 區塊中開啟的任何資料流或檔案。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-125">Use a `finally` block to release resources, for example to close any streams or files that were opened in the `try` block.</span></span>  
  
-   <span data-ttu-id="fc8a1-126">.NET Framework 中的 Managed 例外狀況是在 Win32 結構化例外狀況處理機制上實作的。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-126">Managed exceptions in the .NET Framework are implemented on top of the Win32 structured exception handling mechanism.</span></span> <span data-ttu-id="fc8a1-127">如需詳細資訊，請參閱[結構化例外狀況處理 (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) 和[深入探究 Win32 結構化例外狀況處理的毀損課程 (英文)](http://bytepointer.com/resources/pietrek_crash_course_depths_of_win32_seh.htm)。</span><span class="sxs-lookup"><span data-stu-id="fc8a1-127">For more information, see [Structured Exception Handling (C/C++)](/cpp/cpp/structured-exception-handling-c-cpp) and [A Crash Course on the Depths of Win32 Structured Exception Handling](http://bytepointer.com/resources/pietrek_crash_course_depths_of_win32_seh.htm).</span></span>
