# <a name="compiler-error-cs1001"></a><span data-ttu-id="8ac0a-102">編譯器錯誤 CS1001</span><span class="sxs-lookup"><span data-stu-id="8ac0a-102">Compiler Error CS1001</span></span>
<span data-ttu-id="8ac0a-103">必須是識別項</span><span class="sxs-lookup"><span data-stu-id="8ac0a-103">Identifier expected</span></span>  
  
 <span data-ttu-id="8ac0a-104">您並未提供識別項。</span><span class="sxs-lookup"><span data-stu-id="8ac0a-104">You did not supply an identifier.</span></span> <span data-ttu-id="8ac0a-105">識別項是指您所提供之類別、結構、命名空間、方法、變數等等的名稱。</span><span class="sxs-lookup"><span data-stu-id="8ac0a-105">An identifier is the name of a class, struct, namespace, method, variable and so on that you provide.</span></span>  
  
 <span data-ttu-id="8ac0a-106">下列範例會宣告一個簡單類別，但未提供類別名稱：</span><span class="sxs-lookup"><span data-stu-id="8ac0a-106">The following example declares a simple class but does not give the class a name:</span></span>  
  
```csharp  
//cs1001.cs  
public class              //CS1001  
    {  
        public int Num {get; set;}  
        void MethodA(){}  
    }  
```  
  
 <span data-ttu-id="8ac0a-107">下列範例會產生 CS1001，因為宣告列舉時必須指定成員：</span><span class="sxs-lookup"><span data-stu-id="8ac0a-107">The following sample generates CS1001 because, when declaring an enum, you must specify members:</span></span>  
  
```csharp  
// CS1001.cs  
public class clx  
{  
   enum Colors : int  
   {  
      'a', 'b' // CS1001, 'a' is not a valid int identifier  
       // The following line shows examples of valid identifiers:  
       // Blue, Red, Orange  
   };  
  
   public static void Main()  
   {  
   }  
}  
```  
  
 <span data-ttu-id="8ac0a-108">例如，在介面定義中，即使編譯器不使用參數名稱，仍需要參數名稱。</span><span class="sxs-lookup"><span data-stu-id="8ac0a-108">Parameter names are required even if the compiler doesn't use them, for example, in an interface definition.</span></span> <span data-ttu-id="8ac0a-109">取用介面的程式設計人員需要這些參數，才能了解參數的意義。</span><span class="sxs-lookup"><span data-stu-id="8ac0a-109">These parameters are required so that programmers who are consuming the interface know something about the what the parameters mean.</span></span>  
  
```csharp  
// CS1001-2.cs  
// compile with: /target:library  
interface IMyTest  
{  
   void TestFunc1(int, int);  // CS1001  
   // Use the following line instead:  
   // void TestFunc1(int a, int b);  
}  
  
class CMyTest : IMyTest  
{  
   void IMyTest.TestFunc1(int a, int b)  
   {  
   }  
}  
```
