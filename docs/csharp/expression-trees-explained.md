# <a name="expression-trees-explained"></a><span data-ttu-id="d1a80-104">說明運算式樹狀架構</span><span class="sxs-lookup"><span data-stu-id="d1a80-104">Expression Trees Explained</span></span>

<span data-ttu-id="d1a80-106">運算式樹狀架構是一種定義程式碼的資料結構。</span><span class="sxs-lookup"><span data-stu-id="d1a80-106">An Expression Tree is a data structure that defines code.</span></span> <span data-ttu-id="d1a80-107">其所依據的結構，與編譯器分析程式碼和產生編譯輸出所使用的結構相同。</span><span class="sxs-lookup"><span data-stu-id="d1a80-107">They are based on the same structures that a compiler uses to analyze code and generate the compiled output.</span></span> <span data-ttu-id="d1a80-108">當您閱讀完本教學課程之後，您將會發現運算式樹狀架構與 Roslyn API 中用來建立[分析器和程式碼修正](https://github.com/dotnet/roslyn-analyzers)的類型相當類似</span><span class="sxs-lookup"><span data-stu-id="d1a80-108">As you read through this tutorial, you will notice quite a bit of similarity between Expression Trees and the types used in the Roslyn APIs to build [Analyzers and CodeFixes](https://github.com/dotnet/roslyn-analyzers).</span></span>
<span data-ttu-id="d1a80-109">(分析器和程式碼修正是 NuGet 套件，可在程式碼上執行靜態分析，並為開發人員建議可能的修正)。這些概念很類似，最後都會產生允許以有意義的方式來查看原始程式碼的資料結構。</span><span class="sxs-lookup"><span data-stu-id="d1a80-109">(Analyzers and CodeFixes are NuGet packages that perform static analysis on code and can suggest potential fixes for a developer.) The concepts are similar, and the end result is a data structure that allows examination of the source code in a meaningful way.</span></span> <span data-ttu-id="d1a80-110">不過，運算式樹狀架構是根據一組與 Roslyn API 完全不同的類別和 API。</span><span class="sxs-lookup"><span data-stu-id="d1a80-110">However, Expression Trees are based on a totally different set of classes and APIs than the Roslyn APIs.</span></span>
    
<span data-ttu-id="d1a80-111">讓我們來看一個簡單的範例。</span><span class="sxs-lookup"><span data-stu-id="d1a80-111">Let's look at a simple example.</span></span>
<span data-ttu-id="d1a80-112">程式碼行如下：</span><span class="sxs-lookup"><span data-stu-id="d1a80-112">Here's a line of code:</span></span>
```csharp

```
<span data-ttu-id="d1a80-113">如果您想要以運算式樹狀架構來分析此程式碼，此樹狀會包含數個節點。</span><span class="sxs-lookup"><span data-stu-id="d1a80-113">If you were to analyze this as an expression tree, the tree contains several nodes.</span></span>
<span data-ttu-id="d1a80-114">最外側節點是具有指派的變數宣告陳述式 (`var sum = 1 + 2;`)。該最外側節點包含數個子節點︰變數宣告、指派運算子，以及代表等號右邊的運算式。</span><span class="sxs-lookup"><span data-stu-id="d1a80-114">The outermost node is a variable declaration statement with assignment (`var sum = 1 + 2;`) That outermost node contains several child nodes: a variable declaration, an assignment operator, and an expression representing the right hand side of the equals sign.</span></span> <span data-ttu-id="d1a80-115">此運算式會進一步細分為代表加法運算，以及加法左右運算元的運算式。</span><span class="sxs-lookup"><span data-stu-id="d1a80-115">That expression is further subdivided into expressions that represent the addition operation, and left and right operands of the addition.</span></span>

<span data-ttu-id="d1a80-116">讓我們向下切入以深入了解構成等號右邊的運算式。</span><span class="sxs-lookup"><span data-stu-id="d1a80-116">Let's drill down a bit more into the expressions that make up the right side of the equals sign.</span></span>
<span data-ttu-id="d1a80-117">此運算式為 `1 + 2`。</span><span class="sxs-lookup"><span data-stu-id="d1a80-117">The expression is `1 + 2`.</span></span> <span data-ttu-id="d1a80-118">這是二元運算式。</span><span class="sxs-lookup"><span data-stu-id="d1a80-118">That's a binary expression.</span></span> <span data-ttu-id="d1a80-119">更具體來說，這是二元加法運算式。</span><span class="sxs-lookup"><span data-stu-id="d1a80-119">More specifically, it's a binary addition expression.</span></span> <span data-ttu-id="d1a80-120">二元加法運算式具有兩個子系，分別代表加法運算式的左右節點。</span><span class="sxs-lookup"><span data-stu-id="d1a80-120">A binary addition expression has two children, representing the left and right nodes of the addition expression.</span></span> <span data-ttu-id="d1a80-121">在這裡，這兩個節點都是常數運算式︰左運算元是值 `1`，右運算元是值 `2`。</span><span class="sxs-lookup"><span data-stu-id="d1a80-121">Here, both nodes are constant expressions: The left operand is the value `1`, and the right operand is the value `2`.</span></span>

<span data-ttu-id="d1a80-122">從外表來看，整個陳述式就是一個樹狀︰您可以從根節點開始，然後周遊樹狀中的每個節點，以查看構成陳述式的程式碼：</span><span class="sxs-lookup"><span data-stu-id="d1a80-122">Visually, the entire statement is a tree: You could start at the root node, and travel to each node in the tree to see the code that makes up the statement:</span></span>

- <span data-ttu-id="d1a80-123">具有指派的變數宣告陳述式 (`var sum = 1 + 2;`)</span><span class="sxs-lookup"><span data-stu-id="d1a80-123">Variable declaration statement with assignment (`var sum = 1 + 2;`)</span></span>
    * <span data-ttu-id="d1a80-124">隱含變數類型宣告 (`var sum`)</span><span class="sxs-lookup"><span data-stu-id="d1a80-124">Implicit variable type declaration (`var sum`)</span></span>
        - <span data-ttu-id="d1a80-125">隱含 var 關鍵字 (`var`)</span><span class="sxs-lookup"><span data-stu-id="d1a80-125">Implicit var keyword (`var`)</span></span>
        - <span data-ttu-id="d1a80-126">變數名稱宣告 (`sum`)</span><span class="sxs-lookup"><span data-stu-id="d1a80-126">Variable name declaration (`sum`)</span></span>
    * <span data-ttu-id="d1a80-127">指派運算子 (`=`)</span><span class="sxs-lookup"><span data-stu-id="d1a80-127">Assignment operator (`=`)</span></span>
    * <span data-ttu-id="d1a80-128">二元加法運算式 (`1 + 2`)</span><span class="sxs-lookup"><span data-stu-id="d1a80-128">Binary addition expression (`1 + 2`)</span></span>
        - <span data-ttu-id="d1a80-129">左運算元 (`1`)</span><span class="sxs-lookup"><span data-stu-id="d1a80-129">Left operand (`1`)</span></span>
        - <span data-ttu-id="d1a80-130">加法運算子 (`+`)</span><span class="sxs-lookup"><span data-stu-id="d1a80-130">Addition operator (`+`)</span></span>
        - <span data-ttu-id="d1a80-131">右運算元 (`2`)</span><span class="sxs-lookup"><span data-stu-id="d1a80-131">Right operand (`2`)</span></span>

<span data-ttu-id="d1a80-132">雖然這看起來可能很複雜，其功能卻很強大。</span><span class="sxs-lookup"><span data-stu-id="d1a80-132">This may look complicated, but it is very powerful.</span></span> <span data-ttu-id="d1a80-133">您可以遵循相同的程序來分解較為複雜的運算式。</span><span class="sxs-lookup"><span data-stu-id="d1a80-133">Following the same process, you can decompose much more complicated expressions.</span></span> <span data-ttu-id="d1a80-134">以下列運算式為例：</span><span class="sxs-lookup"><span data-stu-id="d1a80-134">Consider this expression:</span></span>
```csharp
var finalAnswer = this.SecretSauceFunction(
    currentState.createInterimResult(), currentState.createSecondValue(1, 2),
    decisionServer.considerFinalOptions("hello")) +
    MoreSecretSauce('A', DateTime.Now, true);
```

<span data-ttu-id="d1a80-135">上述運算式也是具有指派的變數宣告。</span><span class="sxs-lookup"><span data-stu-id="d1a80-135">The expression above is also a variable declaration with an assignment.</span></span>
<span data-ttu-id="d1a80-136">在此例中，指派的右邊是較為複雜的樹狀。</span><span class="sxs-lookup"><span data-stu-id="d1a80-136">In this instance, the right hand side of the assignment is a much more complicated tree.</span></span>
<span data-ttu-id="d1a80-137">我不要分解此運算式，而是考慮可能會有哪些不同的節點。</span><span class="sxs-lookup"><span data-stu-id="d1a80-137">I'm not going to decompose this expression, but consider what the different nodes might be.</span></span> <span data-ttu-id="d1a80-138">其中包含使用目前物件作為接收器的方法呼叫，其中一個具有明確的 `this` 接收器，另一個則沒有。</span><span class="sxs-lookup"><span data-stu-id="d1a80-138">There are method calls using the current object as a receiver, one that has an explicit `this` receiver, one that does not.</span></span> <span data-ttu-id="d1a80-139">同時包含使用其他接收器物件的方法呼叫，以及不同類型的常數引數。</span><span class="sxs-lookup"><span data-stu-id="d1a80-139">There are method calls using other receiver objects, there are constant arguments of different types.</span></span> <span data-ttu-id="d1a80-140">最後還有一個二元加法運算子。</span><span class="sxs-lookup"><span data-stu-id="d1a80-140">And finally, there is a binary addition operator.</span></span> <span data-ttu-id="d1a80-141">根據 `SecretSauceFunction()` 或 `MoreSecretSauce()` 的傳回型別，該二元加法運算子可能會是已覆寫加法運算子的方法呼叫，或解析為針對某個類別定義之二元加法運算子的靜態呼叫。</span><span class="sxs-lookup"><span data-stu-id="d1a80-141">Depending on the return type of `SecretSauceFunction()` or `MoreSecretSauce()`, that binary addition operator may be a method call to an overridden addition operator, resolving to a static method call to the binary addition operator defined for a class.</span></span>

<span data-ttu-id="d1a80-142">儘管這很複雜，但上述運算式所建立的樹狀就和第一個範例一樣可以輕鬆地巡覽。</span><span class="sxs-lookup"><span data-stu-id="d1a80-142">Despite this perceived complexity, the expression above creates a tree structure that can be navigated as easily as the first sample.</span></span> <span data-ttu-id="d1a80-143">您可以繼續周遊子節點，以尋找運算式中的分葉節點。</span><span class="sxs-lookup"><span data-stu-id="d1a80-143">You can keep traversing child nodes to find leaf nodes in the expression.</span></span> <span data-ttu-id="d1a80-144">父節點都會有其子系的參考，而且每個節點都會有描述節點所屬類型的屬性。</span><span class="sxs-lookup"><span data-stu-id="d1a80-144">Parent nodes will have references to their children, and each node has a property that describes what kind of node it is.</span></span>

<span data-ttu-id="d1a80-145">運算式樹狀架構的結構非常一致。</span><span class="sxs-lookup"><span data-stu-id="d1a80-145">The structure of an expression tree is very consistent.</span></span> <span data-ttu-id="d1a80-146">一旦您了解基本概念，即使是以運算式樹狀架構表示的最複雜程式碼，您也能瞭如指掌。</span><span class="sxs-lookup"><span data-stu-id="d1a80-146">Once you've learned the basics, you can understand even the most complex code when it is represented as an expression tree.</span></span> <span data-ttu-id="d1a80-147">資料結構的簡潔說明 C# 編譯器可如何分析最複雜的 C# 程式，並從該複雜的原始程式碼建立適當的輸出。</span><span class="sxs-lookup"><span data-stu-id="d1a80-147">The elegance in the data structure explains how the C# compiler can analyze the most complex C# programs and create proper output from that complicated source code.</span></span>

<span data-ttu-id="d1a80-148">一旦您熟悉運算式樹狀架構的結構，您將會發現所得到的知識可讓您快速使用許多更進階的案例。</span><span class="sxs-lookup"><span data-stu-id="d1a80-148">Once you become familiar with the structure of expression trees, you will find that knowledge you've gained quickly enables you to work with many more and more advanced scenarios.</span></span> <span data-ttu-id="d1a80-149">運算式樹狀架構的功能很強大。</span><span class="sxs-lookup"><span data-stu-id="d1a80-149">There is incredible power to expression trees.</span></span>

<span data-ttu-id="d1a80-150">除了轉譯要在其他環境中執行的演算法，運算式樹狀架構還可用來更輕鬆地撰寫在執行前檢查程式碼的演算法。</span><span class="sxs-lookup"><span data-stu-id="d1a80-150">In addition to translating algorithms to execute in other environments, expression trees can be used to make it easier to write algorithms that inspect code before executing it.</span></span> <span data-ttu-id="d1a80-151">您可以撰寫其引數為運算式的方法，然後查看這些運算式，再執行程式碼。</span><span class="sxs-lookup"><span data-stu-id="d1a80-151">You can write a method whose arguments are expressions and then examine those expressions before executing the code.</span></span> <span data-ttu-id="d1a80-152">運算式樹狀架構是程式碼的完整表示︰您可以查看任何子運算式的值。</span><span class="sxs-lookup"><span data-stu-id="d1a80-152">The Expression Tree is a full representation of the code: you can see values of any sub-expression.</span></span>
<span data-ttu-id="d1a80-153">您可以查看方法和屬性名稱。</span><span class="sxs-lookup"><span data-stu-id="d1a80-153">You can see method and property names.</span></span> <span data-ttu-id="d1a80-154">您可以查看任何常數運算式的值。</span><span class="sxs-lookup"><span data-stu-id="d1a80-154">You can see the value of any constant expressions.</span></span>
<span data-ttu-id="d1a80-155">您也可以將運算式樹狀架構轉換成可執行委派，再執行程式碼。</span><span class="sxs-lookup"><span data-stu-id="d1a80-155">You can also convert an expression tree into an executable delegate, and execute the code.</span></span>

<span data-ttu-id="d1a80-156">運算式樹狀架構的 API 可讓您建立表示幾乎所有有效程式碼建構的樹狀架構。</span><span class="sxs-lookup"><span data-stu-id="d1a80-156">The APIs for Expression Trees enable you to create trees that represent almost any valid code construct.</span></span> <span data-ttu-id="d1a80-157">不過，為了盡可能保持簡單，您將無法在運算式樹狀架構中建立某些 C# 慣用語。</span><span class="sxs-lookup"><span data-stu-id="d1a80-157">However, to keep things as simple as possible, some C# idioms cannot be created in an expression tree.</span></span> <span data-ttu-id="d1a80-158">其中一個範例是非同步運算式 (使用 `async` 和 `await` 關鍵字)。</span><span class="sxs-lookup"><span data-stu-id="d1a80-158">One example is asynchronous expressions (using the `async` and `await` keywords).</span></span> <span data-ttu-id="d1a80-159">如果您的需求需要非同步演算法，您必須直接操作 `Task` 物件，而不是依賴編譯器支援。</span><span class="sxs-lookup"><span data-stu-id="d1a80-159">If your needs require asynchronous algorithms, you would need to manipulate the `Task` objects directly, rather than rely on the compiler support.</span></span> <span data-ttu-id="d1a80-160">另一個範例是建立迴圈。</span><span class="sxs-lookup"><span data-stu-id="d1a80-160">Another is in creating loops.</span></span> <span data-ttu-id="d1a80-161">通常，您會使用 `for`、`foreach`、`while` 或 `do` 迴圈來建立這些迴圈。</span><span class="sxs-lookup"><span data-stu-id="d1a80-161">Typically, you create these by using `for`, `foreach`, `while` or `do` loops.</span></span> <span data-ttu-id="d1a80-162">如您[在本系列稍後](expression-trees-building.md)所見，運算式樹狀架構的 API 支援單一迴圈運算式，並使用 `break` 和 `continue` 來控制重複迴圈。</span><span class="sxs-lookup"><span data-stu-id="d1a80-162">As you'll see [later in this series](expression-trees-building.md), the APIs for expression trees support a single loop expression, with `break` and `continue` expressions that control repeating the loop.</span></span>
