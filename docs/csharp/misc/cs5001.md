# <a name="compiler-error-cs5001"></a>編譯器錯誤 CS5001

程式未包含適合進入點的靜態 ' Main ' 方法

如果在產生可執行檔的程式碼中找不到具有正確簽章的靜態 `Main` 方法，就會發生這個錯誤。 如果進入點函式（`Main`）定義為錯誤的大小寫（例如，小寫 `main`），也會發生此情況。 如需適用于 `Main` 方法之規則的詳細資訊，請參閱[Main （）和命令列引數](../programming-guide/main-and-command-args/index.md)。

如果 `Main` 方法具有 @no__t 1 修飾詞，請確定[ C#選取的語言版本](../language-reference/configure-language-version.md)是7.1 或更高版本。

只有在編譯可執行檔時（也就是指定[-target： exe](/dotnet/csharp/language-reference/compiler-options/target-exe-compiler-option)或[-target： winexe](/dotnet/csharp/language-reference/compiler-options/target-winexe-compiler-option)編譯器選項）時，才需要 `Main` 方法。 下列 Visual Studio 專案類型預設會指定其中一個選項：

- 主控台應用程式
- ASP.NET Core 應用程式
- WPF 應用程式
- Windows Form 應用程式

## <a name="example"></a>範例

下列範例會產生 CS5001：
  
```csharp
// CS5001.cs
// CS5001 expected when compiled with -target:exe or -target:winexe
public class Program
{
   // Uncomment the following line to resolve.
   // static void Main() {}
}
```
