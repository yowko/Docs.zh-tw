 <span data-ttu-id="770fd-105">下列任何一種情況都可能會造成這個錯誤：</span><span class="sxs-lookup"><span data-stu-id="770fd-105">One scenario that can cause this error is as follows:</span></span>

 <span data-ttu-id="770fd-109">編譯器將這兩種轉換方法視為相等，因為第一個轉換會提供最佳目的類型，而第二個轉換會提供最佳來源類型。</span><span class="sxs-lookup"><span data-stu-id="770fd-109">The compiler will weight the two conversion methods equally, because the first conversion provides the best destination type, and the second conversion provides the best source type.</span></span> <span data-ttu-id="770fd-110">由於編譯器無法選擇，因此會產生這個錯誤。</span><span class="sxs-lookup"><span data-stu-id="770fd-110">Since the compiler will be unable to choose, this error will be generated.</span></span> <span data-ttu-id="770fd-111">若要解決錯誤，請撰寫將 A 轉換為 B 的新明確方法。</span><span class="sxs-lookup"><span data-stu-id="770fd-111">To resolve, write a new explicit method converting A to B.</span></span>

## <a name="example"></a><span data-ttu-id="770fd-113">範例</span><span class="sxs-lookup"><span data-stu-id="770fd-113">Example</span></span>  
 <span data-ttu-id="770fd-114">下列範例會產生 CS0457。</span><span class="sxs-lookup"><span data-stu-id="770fd-114">The following sample generates CS0457.</span></span>  
  
```csharp  
// CS0457.cs  
using System;  
public class A { }  
  
public class G0 {  }  
public class G1<R> : G0 {  }  
  
public class H0 {  
   public static implicit operator G0(H0 h) {  
      return new G0();  
   }  
}  
public class H1<R> : H0 {  
   public static implicit operator G1<R>(H1<R> h) {  
      return new G1<R>();  
   }  
}  
  
public class Test   
{  
   public static void F0(G0 g) {  }  
   public static void Main()   
   {  
      H1<A> h1a = new H1<A>();  
      F0(h1a);   // CS0457  
   }  
}  
```
