<span data-ttu-id="da564-103">可能不正確地指派給了為 using 或 lock 陳述式引數的本機 'variable'。</span><span class="sxs-lookup"><span data-stu-id="da564-103">Possibly incorrect assignment to local 'variable' which is the argument to a using or lock statement.</span></span>  <span data-ttu-id="da564-104">本機原始值會發生 Dispose 呼叫或解除鎖定。</span><span class="sxs-lookup"><span data-stu-id="da564-104">The Dispose call or unlocking will happen on the original value of the local.</span></span>  
  
 <span data-ttu-id="da564-105">有許多情況， `using` 或 `lock` 區塊會導致暫時性的資源流失。</span><span class="sxs-lookup"><span data-stu-id="da564-105">There are several scenarios where `using` or `lock` blocks will result in a temporary leak of resources.</span></span> <span data-ttu-id="da564-106">例如：</span><span class="sxs-lookup"><span data-stu-id="da564-106">Here is one example:</span></span>  
  
 `thisType f = null;`  
  
 `using (f)`  
  
 `{`  
  
 `f = new thisType();`  
  
 `...`  
  
 `}`  
  
 <span data-ttu-id="da564-107">在此例中，當 `thisType` 區塊完成執行，但在區塊內建立的 `using` 物件沒有被記憶體回收時 (雖然最後還是會回收)，會處置變數 `thisType` 的原始值，例如 null。</span><span class="sxs-lookup"><span data-stu-id="da564-107">In this case, the original value, such as null, of the variable `thisType` will be disposed of when the `using` block finishes executing, but the `thisType` object created inside the block will not be, although it will eventually get garbage collected.</span></span>

 `}`

## <a name="example"></a><span data-ttu-id="da564-110">範例</span><span class="sxs-lookup"><span data-stu-id="da564-110">Example</span></span>  
 <span data-ttu-id="da564-111">下列程式碼會產生警告 CS0728。</span><span class="sxs-lookup"><span data-stu-id="da564-111">The following code will generate warning CS0728.</span></span>  
  
```csharp 
// CS0728.cs  
  
using System;  
public class ValidBase : IDisposable  
{  
    public void Dispose() {  }  
}  
  
public class Logger  
{  
    public static void dummy()  
    {  
        ValidBase vb = null;  
        using (vb)   
        {  
            vb = null;  // CS0728  
        }  
        vb = null;  
    }  
    public static void Main() { }  
}  
```
