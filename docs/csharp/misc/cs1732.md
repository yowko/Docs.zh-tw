# 編譯器錯誤 CS1732
預期的參數。  
  
 當 Lambda 運算式的輸入參數後面有逗號，但不指定後續參數時，就會發生這個錯誤。  
  
### 更正這個錯誤  
  
1.  請移除逗號，或加入編譯器希望在逗號後面找到的輸入參數。  
  
## 範例  
 下例會發生 CS1732：  
  
```  
// cs1732.cs // compile with: /target:library class Test { delegate void D(int x, int y); static void Main() { D d = (x,) => { }; // CS1732 } }  
```
