# <a name="compiler-warning-level-1-cs3011"></a>編譯器警告 (層級 1) CS3011
'member': 只有符合 CLS 標準的成員，才可為抽象  
  
 類別成員不可同時為 [抽象](../../csharp/language-reference/keywords/abstract.md) 和不符合 Common Language Specification (CLS) 標準。 CLS 指定應該實作所有類別成員。 如需 cls 符合性的詳細資訊，請參閱[撰寫符合 CLS 標準的程式碼](http://msdn.microsoft.com/en-us/4c705105-69a2-4e5e-b24e-0633bc32c7f3)和[語言獨立性以及與語言無關的元件](../../../docs/standard/language-independence-and-language-independent-components.md)。  
  
## <a name="example"></a>範例  
 下列範例會產生 CS3011：  
  
```  
// CS3011.cs  
  
using System;  
  
[assembly:CLSCompliant(true)]  
public abstract class I  
{  
    [CLSCompliant(false)]  
    public abstract int M();   // CS3011  
  
    // OK  
    [CLSCompliant(false)]  
    public void M2()  
    {  
    }  
}  
  
public class C : I  
{  
    public override int M()  
    {  
        return 1;  
    }  
  
    public static void Main()  
    {  
    }  
}  
```
