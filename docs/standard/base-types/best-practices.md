 <span data-ttu-id="498ce-117">一般而言，規則運算式可以接受兩種類型的輸入：受限制或未受限制。</span><span class="sxs-lookup"><span data-stu-id="498ce-117">In general, regular expressions can accept two types of input: constrained or unconstrained.</span></span> <span data-ttu-id="498ce-118">受限制的輸入是來自已知或可靠來源，並且遵循預先定義格式的文字。</span><span class="sxs-lookup"><span data-stu-id="498ce-118">Constrained input is text that originates from a known or reliable source and follows a predefined format.</span></span> <span data-ttu-id="498ce-119">未受限制的輸入是來自不可靠來源 (例如 Web 使用者) 的文字，且可能未依循預先定義或預期的格式。</span><span class="sxs-lookup"><span data-stu-id="498ce-119">Unconstrained input is text that originates from an unreliable source, such as a web user, and may not follow a predefined or expected format.</span></span>  
  
 <span data-ttu-id="498ce-120">通常撰寫規則運算式模式的目的在於比對有效輸入。</span><span class="sxs-lookup"><span data-stu-id="498ce-120">Regular expression patterns are typically written to match valid input.</span></span> <span data-ttu-id="498ce-121">也就是說，開發人員會檢查要比對的文字，然後撰寫比對該文字的規則運算式模式。</span><span class="sxs-lookup"><span data-stu-id="498ce-121">That is, developers examine the text that they want to match and then write a regular expression pattern that matches it.</span></span> <span data-ttu-id="498ce-122">接著開發人員會利用多個有效的輸入項目進行測試，藉此判斷此模式是否需要更正或進一步詳述。</span><span class="sxs-lookup"><span data-stu-id="498ce-122">Developers then determine whether this pattern requires correction or further elaboration by testing it with multiple valid input items.</span></span> <span data-ttu-id="498ce-123">當模式符合所有假設的有效輸入時，即宣告準備好實際執行，並且可以納入已發行的應用程式中。</span><span class="sxs-lookup"><span data-stu-id="498ce-123">When the pattern matches all presumed valid inputs, it is declared to be production-ready and can be included in a released application.</span></span> <span data-ttu-id="498ce-124">這種方式使得規則運算式模式相當適合比對限制的輸入，</span><span class="sxs-lookup"><span data-stu-id="498ce-124">This makes a regular expression pattern suitable for matching constrained input.</span></span> <span data-ttu-id="498ce-125">不過卻不適合比對未受限制的輸入。</span><span class="sxs-lookup"><span data-stu-id="498ce-125">However, it does not make it suitable for matching unconstrained input.</span></span>  
  
 <span data-ttu-id="498ce-126">若要比對未受限制的輸入，規則運算式必須能夠有效率地處理三種文字：</span><span class="sxs-lookup"><span data-stu-id="498ce-126">To match unconstrained input, a regular expression must be able to efficiently handle three kinds of text:</span></span>
