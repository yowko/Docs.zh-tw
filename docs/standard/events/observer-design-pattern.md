# <a name="observer-design-pattern"></a><span data-ttu-id="97986-102">觀察器設計模式</span><span class="sxs-lookup"><span data-stu-id="97986-102">Observer Design Pattern</span></span>
<span data-ttu-id="97986-103">觀察者設計模式可讓訂閱者向提供者註冊，並且接收通知。</span><span class="sxs-lookup"><span data-stu-id="97986-103">The observer design pattern enables a subscriber to register with and receive notifications from a provider.</span></span> <span data-ttu-id="97986-104">它適合任何需要推入型通知的情節。</span><span class="sxs-lookup"><span data-stu-id="97986-104">It is suitable for any scenario that requires push-based notification.</span></span> <span data-ttu-id="97986-105">這個模式會定義「提供者」(也稱為「主題」或「可預見值」)，以及零個、一個或多個「觀察者」。</span><span class="sxs-lookup"><span data-stu-id="97986-105">The pattern defines a *provider* (also known as a *subject* or an *observable*) and zero, one, or more *observers*.</span></span> <span data-ttu-id="97986-106">觀察者會向提供者註冊，而且只要預先定義的條件、事件或狀態有所變更，提供者就會自動呼叫觀察者的其中一種方法，來通知所有觀察者。</span><span class="sxs-lookup"><span data-stu-id="97986-106">Observers register with the provider, and whenever a predefined condition, event, or state change occurs, the provider automatically notifies all observers by calling one of their methods.</span></span> <span data-ttu-id="97986-107">在這個方法呼叫中，提供者也可以提供目前的狀態資訊給觀察者。</span><span class="sxs-lookup"><span data-stu-id="97986-107">In this method call, the provider can also provide current state information to observers.</span></span> <span data-ttu-id="97986-108">在 .NET Framework 中，觀察者設計模式是透過實作泛型 <xref:System.IObservable%601?displayProperty=nameWithType> 和 <xref:System.IObserver%601?displayProperty=nameWithType> 介面來套用。</span><span class="sxs-lookup"><span data-stu-id="97986-108">In the .NET Framework, the observer design pattern is applied by implementing the generic <xref:System.IObservable%601?displayProperty=nameWithType> and <xref:System.IObserver%601?displayProperty=nameWithType> interfaces.</span></span> <span data-ttu-id="97986-109">泛型型別參數代表提供通知資訊的類型。</span><span class="sxs-lookup"><span data-stu-id="97986-109">The generic type parameter represents the type that provides notification information.</span></span>  
  
## <a name="applying-the-pattern"></a><span data-ttu-id="97986-110">套用模式</span><span class="sxs-lookup"><span data-stu-id="97986-110">Applying the Pattern</span></span>  
 <span data-ttu-id="97986-111">觀察者設計模式適合分散式推入型通知，因為它支援清楚分隔兩種不同的元件或應用程式層，例如資料來源 (商務邏輯) 層和使用者介面 (顯示) 層。</span><span class="sxs-lookup"><span data-stu-id="97986-111">The observer design pattern is suitable for distributed push-based notifications, because it supports a clean separation between two different components or application layers, such as a data source (business logic) layer and a user interface (display) layer.</span></span> <span data-ttu-id="97986-112">當提供者使用回呼將目前的資訊提供給其用戶端時，便會實作這個模式。</span><span class="sxs-lookup"><span data-stu-id="97986-112">The pattern can be implemented whenever a provider uses callbacks to supply its clients with current information.</span></span>  
  
 <span data-ttu-id="97986-113">實作模式需要您提供下列項目：</span><span class="sxs-lookup"><span data-stu-id="97986-113">Implementing the pattern requires that you provide the following:</span></span>  
  
-   <span data-ttu-id="97986-114">提供者或主題，也就是傳送通知給觀察者的物件。</span><span class="sxs-lookup"><span data-stu-id="97986-114">A provider or subject, which is the object that sends notifications to observers.</span></span> <span data-ttu-id="97986-115">提供者是實作 <xref:System.IObservable%601> 介面的類別或結構。</span><span class="sxs-lookup"><span data-stu-id="97986-115">A provider is a class or structure that implements the <xref:System.IObservable%601> interface.</span></span> <span data-ttu-id="97986-116">提供者必須實作單一方法 <xref:System.IObservable%601.Subscribe%2A?displayProperty=nameWithType>，觀察者會呼叫該方法，以接收來自提供者的通知。</span><span class="sxs-lookup"><span data-stu-id="97986-116">The provider must implement a single method, <xref:System.IObservable%601.Subscribe%2A?displayProperty=nameWithType>, which is called by observers that wish to receive notifications from the provider.</span></span>  
  
-   <span data-ttu-id="97986-117">觀察者，也就是接收來自提供者之通知的物件。</span><span class="sxs-lookup"><span data-stu-id="97986-117">An observer, which is an object that receives notifications from a provider.</span></span> <span data-ttu-id="97986-118">觀察者是實作 <xref:System.IObserver%601> 介面的類別或結構。</span><span class="sxs-lookup"><span data-stu-id="97986-118">An observer is a class or structure that implements the <xref:System.IObserver%601> interface.</span></span> <span data-ttu-id="97986-119">觀察者必須實作三種方法，這三種方法全都是由提供者呼叫：</span><span class="sxs-lookup"><span data-stu-id="97986-119">The observer must implement three methods, all of which are called by the provider:</span></span>  
  
    -   <span data-ttu-id="97986-120"><xref:System.IObserver%601.OnNext%2A?displayProperty=nameWithType>，提供新的或目前的資訊給觀察者。</span><span class="sxs-lookup"><span data-stu-id="97986-120"><xref:System.IObserver%601.OnNext%2A?displayProperty=nameWithType>, which supplies the observer with new or current information.</span></span>  
  
    -   <span data-ttu-id="97986-121"><xref:System.IObserver%601.OnError%2A?displayProperty=nameWithType>，通知觀察者發生錯誤。</span><span class="sxs-lookup"><span data-stu-id="97986-121"><xref:System.IObserver%601.OnError%2A?displayProperty=nameWithType>, which informs the observer that an error has occurred.</span></span>  
  
    -   <span data-ttu-id="97986-122"><xref:System.IObserver%601.OnCompleted%2A?displayProperty=nameWithType>，指出提供者已完成傳送通知。</span><span class="sxs-lookup"><span data-stu-id="97986-122"><xref:System.IObserver%601.OnCompleted%2A?displayProperty=nameWithType>, which indicates that the provider has finished sending notifications.</span></span>  
  
-   <span data-ttu-id="97986-123">允許提供者追蹤觀察者的機制。</span><span class="sxs-lookup"><span data-stu-id="97986-123">A mechanism that allows the provider to keep track of observers.</span></span> <span data-ttu-id="97986-124">一般而言，提供者會使用容器物件 (例如 <xref:System.Collections.Generic.List%601?displayProperty=nameWithType> 物件) 保留已訂閱通知的 <xref:System.IObserver%601> 實作參考。</span><span class="sxs-lookup"><span data-stu-id="97986-124">Typically, the provider uses a container object, such as a <xref:System.Collections.Generic.List%601?displayProperty=nameWithType> object, to hold references to the <xref:System.IObserver%601> implementations that have subscribed to notifications.</span></span> <span data-ttu-id="97986-125">基於這個目的使用儲存容器，可讓提供者處理零個到無限多個觀察者。</span><span class="sxs-lookup"><span data-stu-id="97986-125">Using a storage container for this purpose enables the provider to handle zero to an unlimited number of observers.</span></span> <span data-ttu-id="97986-126">觀察者接收通知的順序並未定義；提供者可自由使用任何方法來決定順序。</span><span class="sxs-lookup"><span data-stu-id="97986-126">The order in which observers receive notifications is not defined; the provider is free to use any method to determine the order.</span></span>  
  
-   <span data-ttu-id="97986-127"><xref:System.IDisposable> 實作，可讓提供者在通知完成時移除觀察者。</span><span class="sxs-lookup"><span data-stu-id="97986-127">An <xref:System.IDisposable> implementation that enables the provider to remove observers when notification is complete.</span></span> <span data-ttu-id="97986-128">觀察者會從 <xref:System.IObservable%601.Subscribe%2A> 方法接收 <xref:System.IDisposable> 實作的參考，如此觀察者同樣可以在提供者完成傳送通知之前呼叫 <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> 方法來取消訂閱。</span><span class="sxs-lookup"><span data-stu-id="97986-128">Observers receive a reference to the <xref:System.IDisposable> implementation from the <xref:System.IObservable%601.Subscribe%2A> method, so they can also call the <xref:System.IDisposable.Dispose%2A?displayProperty=nameWithType> method to unsubscribe before the provider has finished sending notifications.</span></span>  
  
-   <span data-ttu-id="97986-129">包含提供者傳送給觀察者之資料的物件。</span><span class="sxs-lookup"><span data-stu-id="97986-129">An object that contains the data that the provider sends to its observers.</span></span> <span data-ttu-id="97986-130">這個物件的類型對應至 <xref:System.IObservable%601> 和 <xref:System.IObserver%601> 介面的泛型類型參數。</span><span class="sxs-lookup"><span data-stu-id="97986-130">The type of this object corresponds to the generic type parameter of the <xref:System.IObservable%601> and <xref:System.IObserver%601> interfaces.</span></span> <span data-ttu-id="97986-131">雖然這個物件可以與 <xref:System.IObservable%601> 實作相同，但它常是不同的類型。</span><span class="sxs-lookup"><span data-stu-id="97986-131">Although this object can be the same as the <xref:System.IObservable%601> implementation, most commonly it is a separate type.</span></span>  
  
> [!NOTE]
>  <span data-ttu-id="97986-132">除了實作觀察者設計模式之外，您可能對探索使用 <xref:System.IObservable%601> 和 <xref:System.IObserver%601> 介面建置的程式庫有興趣。</span><span class="sxs-lookup"><span data-stu-id="97986-132">In addition to implementing the observer design pattern, you may be interested in exploring libraries that are built using the <xref:System.IObservable%601> and <xref:System.IObserver%601> interfaces.</span></span> <span data-ttu-id="97986-133">例如，[Reactive Extensions for .NET (Rx)](https://docs.microsoft.com/previous-versions/dotnet/reactive-extensions/hh242985(v=vs.103)) 是由一組擴充方法和 LINQ 標準序列運算子所組成，可支援非同步程式設計。</span><span class="sxs-lookup"><span data-stu-id="97986-133">For example, [Reactive Extensions for .NET (Rx)](https://docs.microsoft.com/previous-versions/dotnet/reactive-extensions/hh242985(v=vs.103)) consist of a set of extension methods and LINQ standard sequence operators to support asynchronous programming.</span></span>
