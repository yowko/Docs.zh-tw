# <a name="xml-integration-with-relational-data-and-adonet"></a><span data-ttu-id="b5a21-102">XML 與關聯式資料和 ADO.NET 互相整合</span><span class="sxs-lookup"><span data-stu-id="b5a21-102">XML Integration with Relational Data and ADO.NET</span></span>
<span data-ttu-id="b5a21-103">**XmlDataDocument** 類別是 **XmlDocument** 的衍生類別，而且含有 XML 資料。</span><span class="sxs-lookup"><span data-stu-id="b5a21-103">The **XmlDataDocument** class is a derived class of the **XmlDocument**, and contains XML data.</span></span> <span data-ttu-id="b5a21-104">**XmlDataDocument** 的好處是提供關聯式與階層式資料之間的橋樑。</span><span class="sxs-lookup"><span data-stu-id="b5a21-104">The advantage of the **XmlDataDocument** is that it provides a bridge between relational and hierarchical data.</span></span> <span data-ttu-id="b5a21-105">可被結合到 **DataSet**，而且兩種類別都可以隨內含資料而同步變更的即是 **XmlDocument**。</span><span class="sxs-lookup"><span data-stu-id="b5a21-105">It is an **XmlDocument** that can be bound to a **DataSet** and both classes can synchronize changes made to data contained in the two classes.</span></span> <span data-ttu-id="b5a21-106">結合 **DataSet** 的 **XmlDocument** 允許 XML 和關聯式資料互相整合，因而您的資料並不一定要成為 XML 或以關聯式格式表示。</span><span class="sxs-lookup"><span data-stu-id="b5a21-106">An **XmlDocument** that is bound to a **DataSet** allows XML to integrate with relational data, and you do not have to have your data represented as either XML or in a relational format.</span></span> <span data-ttu-id="b5a21-107">兩種您都可以選擇，而不必受限於單一資料表示法。</span><span class="sxs-lookup"><span data-stu-id="b5a21-107">You can do both and not be constrained to a single representation of the data.</span></span>  
  
 <span data-ttu-id="b5a21-108">資料以兩種型式提供的好處在於：</span><span class="sxs-lookup"><span data-stu-id="b5a21-108">The benefits of having data available in two views are:</span></span>  
  
- <span data-ttu-id="b5a21-109">XML 文件的結構化部分可以對應到資料集，因此可以有效的儲存、索引和搜尋。</span><span class="sxs-lookup"><span data-stu-id="b5a21-109">The structured portion of an XML document can be mapped to a dataset, and be efficiently stored, indexed, and searched.</span></span>  
  
- <span data-ttu-id="b5a21-110">轉換、驗證和巡覽可透過以關聯方式儲存的 XML 資料上的游標模型有效達成。</span><span class="sxs-lookup"><span data-stu-id="b5a21-110">Transformations, validation, and navigation can be done efficiently through a cursor model over the XML data that is stored relationally.</span></span> <span data-ttu-id="b5a21-111">有時如果將 XML 儲存成 **XmlDocument** 模型，可以更有效地運用關聯式結構。</span><span class="sxs-lookup"><span data-stu-id="b5a21-111">At times, it can be done more efficiently against relational structures than if the XML is stored in an **XmlDocument** model.</span></span>  
  
- <span data-ttu-id="b5a21-112">**DataSet** 可以儲存 XML 的部分資料。</span><span class="sxs-lookup"><span data-stu-id="b5a21-112">The **DataSet** can store a portion of the XML.</span></span> <span data-ttu-id="b5a21-113">換句話說，您可以使用 **XPath** 或 **XslTransform**，將有興趣的元件和屬性儲存到 **DataSet**。</span><span class="sxs-lookup"><span data-stu-id="b5a21-113">That is, you can use **XPath** or **XslTransform** to store to a **DataSet** only those elements and attributes of interest.</span></span> <span data-ttu-id="b5a21-114">從此，可以對較小、篩選過的資料子集進行變更，然後再將變更傳送到 **XmlDataDocument** 中較大的資料。</span><span class="sxs-lookup"><span data-stu-id="b5a21-114">From there, changes can be made to the smaller, filtered subset of data, with the changes propagating to the larger data in the **XmlDataDocument**.</span></span>  
  
 <span data-ttu-id="b5a21-115">您也可以對從 SQL Server 載入 **DataSet** 的資料執行轉換。</span><span class="sxs-lookup"><span data-stu-id="b5a21-115">You can also run a transform over data that was loaded into the **DataSet** from SQL Server.</span></span> <span data-ttu-id="b5a21-116">另一種選擇是結合 .NET Framework 類別樣式管理的 WinForm 和 WebForm 控制項，成為自 XML 輸入資料流填入的 **DataSet**。</span><span class="sxs-lookup"><span data-stu-id="b5a21-116">Another option is to bind .NET Framework classes-style-managed WinForm and WebForm controls to a **DataSet** that was populated from an XML input stream.</span></span>  
  
 <span data-ttu-id="b5a21-117">除了支援 **XslTransform** 之外，**XmlDataDocument** 還公開關聯式資料給 **XPath** 查詢和驗證。</span><span class="sxs-lookup"><span data-stu-id="b5a21-117">In addition to supporting **XslTransform**, an **XmlDataDocument** exposes relational data to **XPath** queries and validation.</span></span>  <span data-ttu-id="b5a21-118">基本上，關聯式資料和關聯式設備都提供所有的 XML 服務，如控制項結合、Codegen 等等，可以在不危害 XML 真實性的結構化 XML 設計上使用。</span><span class="sxs-lookup"><span data-stu-id="b5a21-118">Basically, all XML services are available over relational data, and relational facilities, such as control binding, codegen, and so on, are available over a structured projection of XML without compromising XML fidelity.</span></span>  
  
 <span data-ttu-id="b5a21-119">因為 **XmlDataDocument** 繼承自 **XmlDocument**，因此會提供 W3C DOM 的實作。</span><span class="sxs-lookup"><span data-stu-id="b5a21-119">Because **XmlDataDocument** is inherited from an **XmlDocument**, it provides an implementation of the W3C DOM.</span></span> <span data-ttu-id="b5a21-120">**XmlDataDocument** 與 **DataSet** 有關，而且在 Dataset 中儲存其資料的子集，但仍然不會限制或更改將它做為 **XmlDocument** 來使用。</span><span class="sxs-lookup"><span data-stu-id="b5a21-120">The fact that the **XmlDataDocument** is associated with, and stores a subset of its data within, a **DataSet** does not restrict or alter its use as an **XmlDocument** in any way.</span></span> <span data-ttu-id="b5a21-121">為使用 **XmlDocument** 而撰寫的程式碼不會隨 **XmlDataDocument** 而變更。</span><span class="sxs-lookup"><span data-stu-id="b5a21-121">Code written to consume an **XmlDocument** works unaltered against an **XmlDataDocument**.</span></span> <span data-ttu-id="b5a21-122">**DataSet** 藉由定義資料表、資料行、關聯和限制，提供相同資料的關聯式檢視，而且是獨立、記憶體中的使用者資料存放區。</span><span class="sxs-lookup"><span data-stu-id="b5a21-122">The **DataSet** provides the relational view of the same data by defining tables, columns, relations, and constraints, and is a stand-alone, in-memory user data store.</span></span>
