## <a name="using-mutexes"></a><span data-ttu-id="040b9-107">使用 Mutex</span><span class="sxs-lookup"><span data-stu-id="040b9-107">Using Mutexes</span></span>  
 <span data-ttu-id="040b9-108">執行緒會呼叫 Mutex 的 <xref:System.Threading.WaitHandle.WaitOne%2A> 方法來要求擁有權。</span><span class="sxs-lookup"><span data-stu-id="040b9-108">A thread calls the <xref:System.Threading.WaitHandle.WaitOne%2A> method of a mutex to request ownership.</span></span> <span data-ttu-id="040b9-109">該呼叫會封鎖起來，直到 Mutex 可供使用或直到選擇性的逾時間隔時間過去。</span><span class="sxs-lookup"><span data-stu-id="040b9-109">The call blocks until the mutex is available, or until the optional timeout interval elapses.</span></span> <span data-ttu-id="040b9-110">如果沒有任何執行緒擁有 Mutex，則 Mutex 的狀態為已發出訊號。</span><span class="sxs-lookup"><span data-stu-id="040b9-110">The state of a mutex is signaled if no thread owns it.</span></span>  
  
 <span data-ttu-id="040b9-111">執行緒會藉由呼叫 Mutex 的 <xref:System.Threading.Mutex.ReleaseMutex%2A> 方法來釋放它。</span><span class="sxs-lookup"><span data-stu-id="040b9-111">A thread releases a mutex by calling its <xref:System.Threading.Mutex.ReleaseMutex%2A> method.</span></span> <span data-ttu-id="040b9-112">Mutex 具有執行緒相似性；也就是說，Mutex 的釋放只能由擁有該 Mutex 的執行緒來執行。</span><span class="sxs-lookup"><span data-stu-id="040b9-112">Mutexes have thread affinity; that is, the mutex can be released only by the thread that owns it.</span></span> <span data-ttu-id="040b9-113">如果執行緒釋放非它擁有的 Mutex，執行緒中就會擲回 <xref:System.ApplicationException>。</span><span class="sxs-lookup"><span data-stu-id="040b9-113">If a thread releases a mutex it does not own, an <xref:System.ApplicationException> is thrown in the thread.</span></span>  
  
 <span data-ttu-id="040b9-114">因為 <xref:System.Threading.Mutex> 類別衍生自 <xref:System.Threading.WaitHandle>，您也可以呼叫 <xref:System.Threading.WaitHandle> 的靜態 <xref:System.Threading.WaitHandle.WaitAll%2A> 或 <xref:System.Threading.WaitHandle.WaitAny%2A> 方法，搭配其他等候控制代碼來要求 <xref:System.Threading.Mutex> 的擁有權。</span><span class="sxs-lookup"><span data-stu-id="040b9-114">Because the <xref:System.Threading.Mutex> class derives from <xref:System.Threading.WaitHandle>, you can also call the static <xref:System.Threading.WaitHandle.WaitAll%2A> or <xref:System.Threading.WaitHandle.WaitAny%2A> methods of <xref:System.Threading.WaitHandle> to request ownership of a <xref:System.Threading.Mutex> in combination with other wait handles.</span></span>  
  
 <span data-ttu-id="040b9-115">如果執行緒擁有 <xref:System.Threading.Mutex>，該執行緒就能在重複的等候要求呼叫中指定相同的 <xref:System.Threading.Mutex>，而不需封鎖其執行；但是，它也必須進行相同次數的 <xref:System.Threading.Mutex> 釋放作業，以釋放擁有權。</span><span class="sxs-lookup"><span data-stu-id="040b9-115">If a thread owns a <xref:System.Threading.Mutex>, that thread can specify the same <xref:System.Threading.Mutex> in repeated wait-request calls without blocking its execution; however, it must release the <xref:System.Threading.Mutex> as many times to release ownership.</span></span>

> [!NOTE]
>  <span data-ttu-id="040b9-120">在 .NET Framework 1.0 和 1.1 版中，會將遭到放棄的 <xref:System.Threading.Mutex> 設為已發出信號的狀態，下一個等候中的執行緒則會取得擁有權。</span><span class="sxs-lookup"><span data-stu-id="040b9-120">In the .NET Framework versions 1.0 and 1.1, an abandoned <xref:System.Threading.Mutex> is set to the signaled state and the next waiting thread gets ownership.</span></span> <span data-ttu-id="040b9-121">如果沒有執行緒在等候，<xref:System.Threading.Mutex> 就會保持已發出信號的狀態。</span><span class="sxs-lookup"><span data-stu-id="040b9-121">If no thread is waiting, the <xref:System.Threading.Mutex> remains in a signaled state.</span></span> <span data-ttu-id="040b9-122">不會有例外狀況擲回。</span><span class="sxs-lookup"><span data-stu-id="040b9-122">No exception is thrown.</span></span>  
  
 <span data-ttu-id="040b9-123">如果是全系統 Mutex，遭到放棄的 Mutex 可能表示應用程式已意外終止 (例如，透過使用「Windows 工作管理員」)。</span><span class="sxs-lookup"><span data-stu-id="040b9-123">In the case of a system-wide mutex, an abandoned mutex might indicate that an application has been terminated abruptly (for example, by using Windows Task Manager).</span></span>
